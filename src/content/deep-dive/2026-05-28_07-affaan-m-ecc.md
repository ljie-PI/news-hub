## 1. 定位与痛点剖析

ECC（affaan-m/ECC）将自己定位为"面向 Agent 的 Harness 原生操作系统"，而不是又一套配置模板。其作者是 Anthropic Hackathon 获奖者，项目源自 10 个月以上的日常生产使用迭代。痛点在于：当前 Claude Code、Codex、Cursor、OpenCode 等 Agent Harness 虽然好用，但开发者很快就会撞上一致的问题——上下文爆炸、记忆不持久、技能碎片、安全审计缺位、子代理无序、跨语言代码评审标准不统一。ECC 用一整套"Skills + Instincts + Memory + Security + Research-first"的方法论，把这些零散的工程化经验系统化、可安装、可演进。

## 2. 核心架构与技术细节

ECC 的内容体量非常庞大：**61 个专精子代理**（planner、architect、tdd-guide、code-reviewer、security-reviewer、各语言 reviewer/build-resolver、mle-reviewer 等）、**246 个 Skills** 工作流、**34 条 Rules**、**14 个 MCP 服务器**、**8 类 Hook 事件**，并附带 76 个 legacy 命令兼容层。仓库结构清晰：`agents/` 子代理、`skills/` 工作流、`rules/` 分语言（common/typescript/python/golang/swift/php/arkts）、`hooks/` 自动化、`scripts/` 跨平台 Node.js 入口、`contexts/` 动态提示注入、`mcp-configs/` MCP 集成。最新 v2.0.0-rc.1（2026 年 4 月）引入了 `ecc_dashboard.py` 桌面看板、`ecc2/` 目录与新的 `dashboard/start/sessions/status/stop/resume/daemon` 命令，并新增 brand-voice、social-graph-ranker、manim-video、remotion-video-creation、nestjs-patterns、prediction-market 等业务向 Skill。v1.8 引入的 `ECC_HOOK_PROFILE=minimal|standard|strict`、`/harness-audit`、`/loop-start`、`/quality-gate`、`/model-route` 等命令，构成 Harness 性能优化的核心抓手。

## 3. 竞品对比与生态站位

围绕 AI 编程 Agent 的"操作系统层"近一年快速膨胀，竞品包括 obra/superpowers（同期热门的 Agent Skill 框架）、anthropics/claude-plugins-official（官方插件目录）、agentmemory（持久化记忆）。ECC 的差异化是**一体化**：把技能、子代理、记忆、安全、Hook 与多模型路由打包成单一可安装系统，并向多个 Harness 同时分发；它还配套 AgentShield 安全审计器（1282 个测试、98% 覆盖率、102 条规则）和商业化路径——OSS 基础版 MIT，ECC Pro 19 美元/座/月覆盖私有仓库与 GitHub App。这种"OSS 引流 + Pro 变现"模式在 Agent 生态中具有示范意义。

## 4. 开发者反馈与局限性

短时间日新增 2k+ Stars 印证了开发者对"Harness OS"的需求，但项目本身也明确警告了若干坑点：插件无法自动分发 `rules/`，如果同时 `/plugin install` 又跑 `install.sh --profile full` 会产生重复 Skill 与 Hook；三套不可互换的命名（`affaan-m/ECC`、`ecc@ecc`、`ecc-universal`）对新手不友好；`multi-*` 命令还依赖额外的 `ccg-workflow` 运行时。庞大体量（246 Skills）也带来上下文与维护负担，需要 `ecc doctor`/`repair` 等诊断命令辅助。
