# Claude Code Pro Max 配额 1.5 小时耗尽：用户大规模投诉

**日期**: 2026-04-13 | **来源**: Hacker News | **票数**: 501 | **评论**: 466

---

## 事件背景

2026 年 4 月，Claude Code 订阅用户（Pro Max，5x 配额）开始在 GitHub Issues 和 HN 大规模报告：即使是"温和使用"，每日配额在 1-2 小时内就被耗尽，剩余时间无法继续工作。这与 Anthropic 官方宣传的"5 倍于标准配额"严重不符。该 GitHub Issue（#45756）积累了 501 票和 466 条评论，成为 Claude Code 仓库历史上最受关注的 issue 之一。

这一事件与 #46829（Anthropic 悄然将缓存 TTL 从 1 小时降至 5 分钟）密切相关，二者共同揭示了 Anthropic 在 3 月初的一次未经公告的基础设施变更引发的系统性问题。

---

## 核心观点/产品机制

**用户反映的核心问题：**
- Pro Max 订阅（每月约 $100）承诺的"5x 配额"在实际使用中名不副实
- 简单任务（如询问时间、代码补全）消耗了不成比例的配额
- Claude Code 似乎在后台（甚至在电脑睡眠时）也在消耗配额
- 用户无法看到实时配额消耗明细，透明度几乎为零

**根本原因（结合 #46829 分析）：**
Anthropic 在 3 月 6 日前后将 Claude Code 的提示缓存 TTL（存活时间）从 1 小时静默降至 5 分钟。这意味着：
- 每次对话暂停超过 5 分钟，整个缓存上下文过期
- 下一次请求必须重新上传完整上下文，按缓存写入价（12.5 倍于缓存读取价）计费
- 对于长时间编程会话（主要使用场景），这创造了"缓存重建惩罚"的复合效应
- 结果：相同的工作量消耗了 20-30% 更多的配额/费用
