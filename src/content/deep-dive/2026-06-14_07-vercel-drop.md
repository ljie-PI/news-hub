## 一、产品速览

Vercel Drop 是 Vercel 在 vercel.com/drop 推出的"零配置部署"入口，主打一句口号：**Drop it. It's live.**——把文件、文件夹或 .zip 拖进浏览器，选定团队与项目名，点击 Deploy，几秒内就能拿到生产环境 URL。整个过程无需 Git 仓库、无需安装 Vercel CLI，也不需要本地 Node 环境，是 Vercel 在"开发者门槛"上的一次极致瘦身。

## 二、核心体验：把部署降级为一次拖拽

Drop 的核心是把传统的 `git push → CI → CDN` 流水线折叠成单次浏览器交互。用户拖入静态站点目录或前端构建产物（HTML/CSS/JS、SPA 打包文件、Hugo/Astro 的 `dist`），Vercel 会自动识别入口、上传到 Edge Network 并分配 `*.vercel.app` 域名。由于直接走 Vercel 的 Production 通道，CDN 缓存、HTTPS、自动 HTTP/2 与全球边缘节点开箱即用，体验与 CLI/Git 流程产物完全一致。

## 三、典型使用场景

最直接的受益者是**黑客松与课堂演示**：选手在结束前 30 秒还能把 `dist` 拖上去拿到可分享链接；同样适合**设计师交付静态原型**、**写作者发布单页作品集**，以及**AI 工具生成的一次性页面**——许多 LLM 代码助手输出的就是一个 HTML 文件夹，Drop 让"生成即上线"成为可能。对于团队内部，它也是给非技术同事演示 UI 改动的轻量通道。

## 四、对比 Netlify Drop 与 Cloudflare Pages

这类"拖拽部署"并非首创。**Netlify Drop**（app.netlify.com/drop）是品类开山之作，2015 年左右就已存在，胜在历史悠久、表单/Functions 生态成熟，但近年迭代节奏放缓。**Cloudflare Pages** 提供 `wrangler pages deploy` 和 Direct Upload，强项是免费额度大、与 Workers/R2 深度集成，但浏览器原生拖拽体验不如前两者顺滑。Vercel Drop 的差异点在于：**直接进 Production 而非 Draft URL**、与 Next.js/Edge Functions 同一基础设施、并可在部署后一键"Claim"到正式项目继续用 Git 接管——这条"从一次性 demo 平滑升级到长期项目"的路径是 Netlify Drop 缺少的。

## 五、价值判断

Vercel Drop 本身不是技术突破，而是一次精准的**入口设计**：把新用户首次成功部署的时间压到 10 秒以内，再用"Claim 项目"把流量沉淀到正式工作流。在 AI 生成代码爆发的当下，这种"无账号摩擦的发布通道"会成为 PaaS 厂商争夺生成式开发者的关键触点，值得 Netlify 与 Cloudflare 正面回应。
