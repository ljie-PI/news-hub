# 部署了 20 个 Agent 后，我们才承认根本不知道公司里有多少个 Agent

## 事件背景
一位从业者在 r/AI_Agents 讲述了团队在做安全审查时的尴尬发现：当他们尝试盘点公司内部的 Agent 时，翻出了一批「被建好、被部署、又被遗忘」的 Agent。这些 Agent 的搭建者早已换岗，没有负责人、没有文档，却仍在运行，且依然持有生产环境凭证。更棘手的是，没人说得清哪些 Agent 能访问客户数据。

## 核心观点
作者点出企业 AI 落地中一个被低估的风险：Agent 蔓延（agent sprawl）。当每个人都能快速搭 Agent 并接入生产系统时，治理却没跟上。缺乏统一清单、所有权和权限审计，意味着安全边界形同虚设——一个被遗忘却仍在运行的 Agent，就是一个持有真实凭证的隐形攻击面。

## 社区热议与争议点
评论区观点分化（非逐字引用）。一派认为这是「影子 IT」在 AI 时代的重演，主张建立 Agent 注册表、强制标注 owner 与数据访问范围、定期轮换凭证。另一派则更悲观，认为在缺乏平台约束前，任何清单都会迅速过期。也有人提出用最小权限、短期令牌和集中式网关来从架构上根治，而非靠事后盘点。

## 行业影响与未来展望
随着企业 Agent 数量激增，AgentOps 与身份治理正成为刚需。可以预见的方向包括：Agent 身份即服务、统一的凭证与权限管控平面，以及把 Agent 纳入现有的安全与合规审计流程。谁能先把「可见性」和「可问责」做实，谁就能真正把 Agent 从风险变成资产。
