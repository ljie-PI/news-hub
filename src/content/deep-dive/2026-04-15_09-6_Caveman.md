# 6. Caveman — Why use so many token when few do trick?

**Product Hunt 排名：#6 | 票数：207 | GitHub Stars：24.9K | 分类：Open Source / Developer Tools / AI**

---

## 事件背景

随着 AI 编程助手（Claude Code、Cursor、Windsurf 等）的普及，一个新的成本问题浮现：AI 输出极其冗余。Claude 等模型在回复时喜欢详细解释、列举步骤、重复确认，在长对话中这种冗余会导致 Token 消耗急速膨胀，直接影响使用成本和响应速度。Caveman 以一个极具幽默感的名字和 Slogan——"Why use so many token when few do trick?"——横空出世。

## 核心观点 / 产品机制

Caveman 是一个提示词插件（CLAUDE.md / system prompt 级别的规则集），通过在 Claude Code、Cursor、Windsurf、GitHub Copilot 等工具中一行安装，强制 AI 使用最精简的方式回复。核心机制：

- **四个"咕哝级别"（Grunt Levels）**：从轻度精简到极度简洁，用户可按需调整
- **简洁 Commit 信息**：提交消息遵守约定式提交格式，不废话
- **一行 PR Review**：代码审查只输出关键意见，不写长篇论文
- **输入压缩（Input Compression）**：同时压缩发送给 AI 的上下文，双向节省 Token

声称可削减约 75% 的输出 Token，在保持技术准确性的前提下。

## 社区热议与争议点

24.9K GitHub Stars 说明这个项目在开发者社区已经广泛传播。PH 上的讨论：

- **真实效果验证**：多位用户分享了对比数据，证实 Token 节省效果明显，但也有人指出"有时 AI 过于简洁反而需要追问，来回更多"。
- **安全性考量**：有人担心系统级提示词插件可能影响 AI 在安全关键场景下的谨慎性。
- **哲学之争**：有人认为这是"治标不治本"——真正的解法是模型本身学会精简，而不是通过外部约束。

## 行业影响与未来展望

Caveman 的爆红（24.9K stars）揭示了一个被低估的需求：AI 工具成本管理。随着 AI 使用量增加，Token 经济学将越来越重要。这个插件简单粗暴地解决了问题，但也预示着未来 AI 开发工具需要内置成本控制机制，而不是让用户自己装插件来补救。这是 Anthropic 等模型厂商需要正视的用户反馈。
