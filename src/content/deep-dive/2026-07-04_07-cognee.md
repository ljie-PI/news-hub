# topoteretes/cognee

## 定位与痛点剖析
大模型智能体天生健忘，跨会话难以留存长期记忆。Cognee 定位为开源的 AI 记忆平台，为智能体提供可自托管的持久长期记忆。它把任意格式的数据摄入后构建知识图谱，让每个智能体都能带着完整上下文去回忆、关联与行动，缓解传统 RAG 只能按相似度召回、缺乏关系推理的痛点。

## 核心架构与技术细节
Cognee 融合向量嵌入、图推理与基于认知科学的本体生成，使文档既能按语义检索，又能按会演化的关系相互连接。对外暴露 remember、recall、forget、improve 四个核心操作。1.0 版本的一大亮点是可把整套记忆层收敛到单个 Postgres 实例：关系用 Postgres 图后端、向量用 pgvector、会话缓存用 SQL，替代 Neo4j 加向量库加 Redis 的传统多服务栈。它同时提供 CLI、MCP 服务器与 Docker 镜像。

## 竞品对比与生态站位
相较 Mem0、Zep 等记忆方案，Cognee 更强调知识图谱与本体的显式建模，而非单纯的向量存储。它可作为 Claude Code 插件注入跨会话记忆，也提供 TypeScript、Rust 客户端与托管的 Cognee Cloud，整体生态覆盖较广。

## 开发者反馈与局限性
官方自报口径称在其 CI 基准中 Postgres 方案比图加向量的分离部署快约 10%，但此为自测数据需谨慎看待。项目仍在快速迭代，构建知识图谱高度依赖 LLM 调用，成本与延迟不容忽视；`cognee-cli -ui` 启动的 MCP 服务器需 Docker 运行时，对本地环境有一定要求。
