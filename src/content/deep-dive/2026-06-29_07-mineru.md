# MinerU

## 定位
MinerU 由上海人工智能实验室 OpenDataLab 出品，孵化于 InternLM 大模型预训练的数据建设需求，旨在把 PDF、图片及 DOCX/PPTX/XLSX 等复杂文档转换为 Markdown、JSON 等机器可读格式，专门解决科技文献的符号转换难题，服务于 RAG 检索、信息抽取与 Agent 工作流。项目目前已积累约 7.1 万 Star、6 千 Fork，社区活跃。3.1 版起授权协议从 AGPLv3 改为基于 Apache 2.0 的自定义 MinerU 许可，大幅降低商用与社区集成门槛，是国产开源文档解析的代表作。

## 架构
核心能力包括版式分析、阅读顺序还原、去除页眉页脚页码、公式转 LaTeX、表格转 HTML，OCR 覆盖 109 种语言并自动识别扫描件。提供三种推理后端：pipeline（纯 CPU 可跑、快速无幻觉，OCR 已升级至 PP-OCRv6）、vlm-engine（基于 MinerU2.5-Pro-1.2B 视觉模型，兼容 vLLM/LMDeploy/mlx，高精度）与 hybrid 混合后端（原生文本抽取、低幻觉）。配套 mineru-api、mineru-router 支持多 GPU 一键部署与任务负载均衡，滑动窗口机制使万页长文档稳定解析，并适配昇腾、寒武纪、摩尔线程、昆仑芯等国产芯片，支持 Win/Linux/Mac。

## 竞品
直接对手有 Marker、Docling、unstructured 及百度 PaddleOCR 的 PP-StructureV3，加上 Textract、Mathpix 等商用 API。MinerU 优势在于开源彻底、CPU 友好、中文与公式表格识别能力强，pipeline 后端在 OmniDocBench 达 86.2 分；官方亦坦言相较成熟商用产品仍属年轻。

## 反馈
社区热度高，被 LangChain、LlamaIndex、RAGFlow、Dify、FastGPT 等广泛集成，并提供 MCP Server 接入 Cursor、Claude Desktop。用户称赞解析精度与离线部署能力，但复杂版式、扫描件、手写体仍偶有偏差，GPU 显存占用与安装环境也是常见门槛。
