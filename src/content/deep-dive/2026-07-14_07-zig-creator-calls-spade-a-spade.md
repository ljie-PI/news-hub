# Zig Creator Calls Spade a Spade, Anthropic Blows Smoke

**原帖（HN 讨论）**: https://news.ycombinator.com/item?id=48889637
**原文**: https://raymyers.org/post/zed-creator-calls-spade-a-spade/

## 事件背景

Anthropic 旗下的 Bun 团队本周发布了一篇博客，解释为何将 JavaScript 运行时 Bun 从 Zig 语言移植到 Rust——值得注意的是，这篇"解释"是在迁移合入主线两个月之后才发出的。作者 Ray Myers（曾任某编码 agent 创业公司首席架构师，既是 Anthropic 模型客户又是 Claude Code 竞品）指出，这种延后发文的时机恰好让"Anthropic 用 AI 以极快速度完成 Rust 重写"的营销式头条先行传播。Zig 语言创造者 Andrew Kelley 随后发布了一篇措辞异常直白、近乎人身攻击的回应文，引爆社区讨论。

## 核心观点

Myers 的核心主张是：Anthropic 是一个"不可靠的叙述者"。它已融资 1320 亿美元、估值逼近万亿 IPO，却尚未盈利，因此必须兜售"软件工程即将消失、编码先消失、然后是大多数人类劳动"这一宏大叙事来支撑估值。Bun 的 Rust 重写被包装成"AI 速度奇迹"，正是这套叙事的一环。他呼吁开发者戴上怀疑的帽子、独立思考，警惕基于恐惧（裁员焦虑、"被时代抛下"）而非理性做出的架构与人事决策。

## 社区热议与争议点

评论区呈现明显对立。**反方（认同 Zig 那派多为情绪化）**：如 vlaaad 认为"我们读的是同一篇文章吗？Anthropic 的博客充满技术细节、切实改进了产品，而 Zig 的回应是酸味十足、满是人身攻击的意见稿"；manojlds 补充 Bun 和 Zig 根本不在同一市场竞争。**正方（认同 Myers）**：LAC-Tech 称围绕 Andrew 言论的愤怒是"表演式、夸张的"，整件事"散发着公关噱头的味道"——展示你能烧掉大把 token 去 vibe code 一次重构。也有人质疑作者"我用 Codex"的声明能否证明其无偏见，matsemann 指出重度使用 AI agent 的人本身就带立场。

## 行业影响与未来展望

这场争论折射出 AI 泡沫期的深层焦虑：当巨额资本绑定"AI 终结软件工程"的叙事时，技术选型（如换语言）极易被裹挟为营销素材。长远看，社区正在形成一种反制声音——可以采用编码 agent，但不必把"黑箱软件工厂"奉为理想。Myers 预告将在 Software Should Work 大会上给出语言设计方向。
