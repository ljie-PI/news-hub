# remotion-dev/remotion — 用 React 代码制作视频

## 1. 定位与痛点剖析

Remotion 是一个**用 React 编写视频的框架**，核心理念是：视频 = 带时间轴的 React 组件。

解决的痛点：

- **传统视频制作工具（Premiere、After Effects）**：手动拖拽、不可编程、无法批量生成、版本控制困难
- **数据驱动视频需求**：报表可视化、个性化视频（如 GitHub Unwrapped 年终总结）、动态内容视频，用传统工具几乎无法实现规模化
- **Web 开发者的技能迁移壁垒**：前端开发者掌握 CSS、Canvas、SVG、WebGL，但这些技能无法直接用于视频生产

Remotion 让前端开发者用已有技能（React + Web APIs）制作专业级视频，并且可以通过编程批量生成个性化视频内容。

## 2. 核心架构与技术细节

**技术栈**：基于 Node.js + Puppeteer/Chrome headless 渲染，每一帧实际上是截图合并。

**核心概念**：
- `<Composition>`：定义视频的宽高、帧率、时长
- `useCurrentFrame()`：返回当前帧号，基于此驱动所有动画
- `interpolate()`：帧范围到值范围的插值映射（类似 CSS animation 的 keyframe）
- `<Sequence>`：时间轴上的片段容器，支持嵌套

**渲染方案**：
- **本地 CLI**：`npx remotion render` 本地渲染
- **Node.js API**：嵌入到后端服务，编程式触发渲染
- **Lambda 渲染**：AWS Lambda 并行渲染大幅提速（付费功能相关）
- **Cloud Run**：Google Cloud Run 渲染

**生态能力**：
- `@remotion/player`：在浏览器中实时预览视频（React 组件）
- `@remotion/captions`：字幕生成
- `@remotion/three`：Three.js 3D 场景集成
- `@remotion/gif`：GIF 生成
- 支持 Tailwind CSS、Lottie 动画等前端生态

**典型案例**：
- Fireship 频道视频（YouTube 知名科技频道）
- GitHub Unwrapped（用户年度报告个性化视频）

## 3. 竞品对比与生态站位

| 工具 | 编程驱动 | Web 技术 | 批量生成 | 学习曲线 |
|------|---------|---------|---------|---------|
| **Remotion** | ✅ React | ✅ 完整 CSS/Canvas/WebGL | ✅ | 低（对前端开发者） |
| Motion Canvas | ✅ TypeScript | 部分 | ✅ | 中 |
| Manim | ✅ Python | ❌ | ✅ | 高 |
| After Effects | ❌ 手动 | ❌ | 有限 | 高 |
| HeyGen/Synthesia | ❌ | ❌ | ✅（AI 生成） | 低 |

Remotion 是"Web 开发者制作视频"的最佳路径，特别适合需要**数据驱动个性化视频**和**品牌动态内容**的场景。

## 4. 开发者反馈与局限性

**优势**：
- 对 React 开发者零学习成本，用现有 Web 技术即可
- Fast Refresh 支持实时预览，开发体验好
- 生态丰富，社区活跃（Discord 万人+）

**局限性**：
- **特殊许可证**：商业使用需购买公司授权（非纯 MIT），小型公司/个人免费，但企业使用需付费
- 渲染速度：本地逐帧截图渲染，长视频渲染耗时较长（Lambda 渲染可解决但需成本）
- 复杂的时间轴逻辑（多图层同步）比专业视频编辑软件难管理
- 不支持直接导入现有视频素材做非线性编辑（偏向程序化生成，非编辑工具）
