# safishamsi/graphify

## 定位与痛点剖析
Graphify 是一款面向 AI 编程助手的"技能"插件，核心痛点是大型代码库中助手靠逐文件 grep 检索效率低、上下文割裂。它把整个项目——代码、SQL 架构、R 脚本、Shell 脚本、文档、PDF、图片乃至视频——映射为可查询的知识图谱，让助手直接向图谱提问，而非反复翻阅零散文件。对于动辄数十万行、跨多种语言与格式的巨型仓库，人工建立全局认知几乎不可能，图谱化正是为替代这种碎片化理解而生。

## 核心架构与技术细节
项目基于 Python 3.10 以上实现，推荐用 uv 或 pipx 隔离安装，官方包名为 graphifyy（双 y），命令行仍是 graphify。运行 /graphify . 后产出三件套：可交互点选的 graph.html、汇总要点的 GRAPH_REPORT.md 与完整的 graph.json。图谱一次构建即可反复查询，无需重读源文件；另有 callflow-html 导出可生成带 Mermaid 调用流图的架构页。它通过 Git 钩子把解释器路径写入脚本，使提交钩子在图形客户端与持续集成中也能触发增量更新。

## 竞品对比与生态站位
相较 Sourcegraph、GitHub 代码搜索等偏纯检索的方案，Graphify 强调与助手工作流深度耦合，据自报口径支持 Claude Code、Codex、Cursor、Gemini CLI、Copilot 等二十余个平台，走跨框架技能路线，生态覆盖面较广。它把知识图谱当作助手的长期记忆层，弱化了对单一向量检索的依赖。

## 开发者反馈与局限性
项目挂着 YC S26 标签，星标数官方宣称为 77072 且单日增长近千，热度迅猛。局限在于安装易踩坑：环境变量未配置会报命令找不到，苹果与视窗系统直接用 pip 安装常导致模块缺失，PowerShell 下须用不带斜杠的写法。官方建议改用隔离安装并在升级后重装钩子以保证解释器路径正确。
