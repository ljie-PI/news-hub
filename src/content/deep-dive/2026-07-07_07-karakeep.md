# karakeep-app/karakeep

## 定位与痛点剖析
Karakeep（前身 Hoarder）是一款自托管的「万物收藏」应用，可保存链接、笔记、图片和 PDF，并带 AI 自动打标签与全文搜索。它瞄准 data hoarder 群体：那些在手机上刷 Reddit、Twitter、HackerNews 时随手囤积大量链接、事后却难以检索归类的人。作者自述曾用 Pocket 和 memos，但前者非自托管、后者缺少链接预览与自动标签，于是自建此项目。面向重视数据自主、想把 read-it-later 场景搬进自己家庭服务器的自托管爱好者。

## 核心架构与技术细节
技术栈以 TypeScript 为主，Web 端用 NextJS（app router），数据库与迁移用 Drizzle ORM，认证用 NextAuth，前后端通信用 tRPC，抓取链接用 Puppeteer，AI 能力接 OpenAI 且支持本地 ollama 模型，全文搜索用 Meilisearch。功能上支持规则引擎、图片 OCR、RSS 自动收藏、yt-dlp 视频归档，并用 monolith 做整页存档以对抗 link rot。配套 Chrome/Firefox/Safari 扩展、iOS 与 Android App、REST API 及 CLI，还能经 floccus 与浏览器书签同步。许可为 AGPL-3.0。

## 竞品对比与生态站位
README 的 Alternatives 一节直接点名多个竞品：memos（不做链接归档与预览）、商业产品 mymind、不可自托管的 raindrop，以及 Pocket（已停运）、Linkwarden、Wallabag、Shiori。Karakeep 的差异化在于「自托管优先 + AI 自动标签 + 整页归档」的组合，填补了纯书签管理器与笔记应用之间的空档。

## 开发者反馈与局限性
仓库 open issues 高达约 659 个，README 也自承「under heavy development」。近期 issue 暴露若干安全与数据问题：OAuth 邮箱账号链接导致的账户接管、OAuth 绕过强制邮箱验证、导入 worker 误用空 SQLite 库报 no such table、书签生成重复附件等。功能上仍缺离线阅读与语义搜索（列为 Planned）。安全相关 issue 提示自托管部署需谨慎配置。
