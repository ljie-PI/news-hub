# cjpais/Handy

## 定位与痛点剖析
Handy 是一款免费、开源、可扩展的语音转文字桌面应用，主打完全离线运行。它瞄准两类痛点：一是隐私顾虑，多数 STT 工具需把音频上传云端；二是缺乏真正可自定义的开源方案。作者明确表示 Handy 不追求做「最准确」的应用，而是要做「最易 fork」的那一个，为开发者提供可二次开发的本地语音工具基座。

## 核心架构与技术细节
Handy 基于 Tauri v2 构建，采用 Rust 后端 + React/TypeScript 前端的清晰分层。工作流是：一个可配置快捷键触发录音（按下录音、松开转写），Silero VAD 过滤静音，再由 Whisper 或 Parakeet V3 完成本地推理，结果自动粘贴到当前聚焦的文本框。它通过 whisper-rs、transcription-rs 做本地推理并在支持时启用 GPU 加速，还提供 CLI 参数、Unix 信号处理与单实例 IPC 模式。

## 竞品对比与生态站位
相比云端 STT 与闭源听写工具，Handy 以离线、隐私和 MIT 许可的可扩展性取胜；跨 macOS、Windows、Linux。Whisper 与 Parakeet 双后端并存，兼顾 GPU 精度与 CPU-only 效率。

## 开发者反馈与局限性
项目 star 约 26k、fork 约 2.3k（自报口径），被开发者称为「最易 fork 的语音转文字应用」，其干净的 Rust/TypeScript 分层、双转写后端、CLI 外部控制与单实例 IPC 模式，被视为构建系统级 AI 应用的教学范本。主要局限在于：Linux 平台启动依赖 gtk-layer-shell 等运行时库，缺失时会加载失败并偶发崩溃，需按发行版手动安装；此外由于「可 fork」优先于「最准确」，其转写精度并非首要目标，开箱效果未必领先专用商业方案。
