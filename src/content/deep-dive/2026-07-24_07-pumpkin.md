## 一、定位与痛点

Pumpkin 是一个完全用 Rust 编写的 Minecraft 服务端，口号是「Empowering everyone to host fast and efficient Minecraft servers」。传统官方服务端与主流的 Spigot/Paper 均基于 Java，长期受 GC 停顿、单线程主循环与高内存占用困扰，大型服务器 tick 卡顿严重。Pumpkin 试图用 Rust 的零成本抽象与无 GC 特性，从底层解决性能与内存问题。

## 二、核心架构与技术细节

仓库 100% 由 Rust 构成，采用高度模块化的 workspace 拆分：`pumpkin-protocol`（协议编解码）、`pumpkin-world`（世界与区块）、`pumpkin-inventory`、`pumpkin-nbt`、`pumpkin-registry` 等 crate 各司其职，并通过 `pumpkin-api-macros`、`pumpkin-plugin-api` 及 WIT 插件接口（`pumpkin-plugin-wit`）提供扩展能力。设计目标涵盖多线程并行以榨取性能、同时兼容最新 Java 与 Bedrock 版本、遵循原版机制，配置采用 TOML。已实现区块加载/保存（Vanilla、Linear、Pump 三种格式）、光照、实体生成、RCON、Velocity/BungeeCord 代理等特性。

## 三、竞品对比与生态站位

相比 Paper、Purpur 等 Java 生态成熟方案，Pumpkin 最大差异是语言层面的性能红利与更低资源占用，且原生兼顾 Java 与 Bedrock 双端。与同为 Rust 的 Valence（偏框架/库）相比，Pumpkin 更强调做一个开箱可用、贴近原版的完整服务端。

## 四、开发者反馈与局限性

项目已积累约 7.9k star、553 fork、2438 次提交（自报口径），Discord 社区活跃。但官方明确标注「currently under heavy development」，尚未发布 1.0.0，实体 AI、Redstone、Mob/动物等仍为 W.I.P，插件生态与原版行为完整性还需打磨，暂不建议生产环境使用。
