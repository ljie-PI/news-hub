# 01 · NousResearch/hermes-agent

> ⭐ 52,996 stars / week | 84,352 total | 2026-04-15

## a. 定位与痛点剖析

**hermes-agent** 是 Nous Research 推出的自我进化型 AI Agent，核心定位是"会成长的 Agent"。它直指当前 AI Agent 生态最核心的三大痛点：

1. **跨会话失忆**：主流 AI 工具（Claude Code、ChatGPT等）每次对话都从零开始，无法积累用户偏好、项目上下文或已习得的技能；
2. **平台绑定**：大多数 Agent 工具与特定模型绑定，切换成本极高；
3. **不可移植**：大多数方案只能在本地笔记本运行，无法在 VPS、GPU 集群或 Serverless 环境之间自由迁移。

hermes-agent 的核心差异在于其**闭环学习系统**：Agent 在执行复杂任务后会自动创建 Skill 文件，Skill 在使用过程中自我改进，同时通过 FTS5 会话搜索与 LLM 摘要实现跨会话记忆检索，并借助 Honcho dialectic user modeling 构建深度用户画像。

## b. 核心架构与技术细节

**技术栈**：Python（uv 管理依赖）、支持 Python 3.11+、MIT 协议

**核心架构层次**：
- **Agent Loop**：标准 ReAct 循环，内置 40+ 工具（文件、终端、浏览器、图像等）
- **记忆层**：
  - 短期：基于文件的 MEMORY.md + daily notes 体系
  - 长期：FTS5 全文检索（SQLite）+ LLM 摘要 + Honcho 用户建模
  - 技能层：Skill 文件系统，兼容 agentskills.io 开放标准
- **网关层**：统一消息网关，支持 Telegram/Discord/Slack/WhatsApp/Signal/Email，单进程处理多平台
- **执行后端**：六种终端模式（本地/Docker/SSH/Daytona/Singularity/Modal），其中 Daytona 和 Modal 支持 Serverless 休眠/唤醒
- **RL 研究管道**：内置 Atropos RL 环境，可生成轨迹数据用于训练下一代工具调用模型
- **模型无关**：支持 Nous Portal/OpenRouter(200+)/Xiaomi MiMo/GLM/Kimi/MiniMax/HuggingFace/OpenAI 等，`hermes model` 命令一键切换

**安装**：`curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash`，支持 Linux/macOS/WSL2/Android Termux

**迁移支持**：内置 `hermes claw migrate`，可从 OpenClaw 完整迁移配置、记忆、技能、API key

## c. 竞品对比与生态站位

| 工具 | 记忆持久化 | 多平台通信 | 技能自学 | 模型无关 | 自托管 |
|------|-----------|-----------|---------|---------|--------|
| hermes-agent | ✅ 闭环学习 | ✅ 6平台 | ✅ 自动创建 | ✅ 200+模型 | ✅ |
| OpenClaw | ✅ 文件系统 | ✅ 多平台 | ✅ 手动 | ✅ | ✅ |
| goose (AAIF) | ⚠️ 部分 | ❌ 仅CLI/App | ❌ | ✅ 15+模型 | ✅ |
| Claude Code | ❌ | ❌ | ❌ | ❌(仅Claude) | ❌ |

hermes-agent 的生态野心极大：它不只是一个工具，更试图成为**开源 Agent 标准**的制定者（agentskills.io），并提供从 OpenClaw 迁移路径，直接对标市场占有率最高的自托管 Agent 框架。

## d. 开发者反馈与局限性

**正面**：
- 一周内 53k star 是近年开源 Agent 项目最强爆发之一，说明社区对"真正有记忆的 Agent"需求极强
- 支持 Android Termux 扩展了使用场景
- RL pipeline 吸引了大量 AI 研究者

**局限性**：
- 闭环学习的效果高度依赖底层模型能力，弱模型下 Skill 自动生成质量存疑
- Serverless 部署（Daytona/Modal）对国内网络环境不友好
- 刚发布，大量功能文档不完善，社区 issue 快速堆积
- Windows 原生不支持（需 WSL2）
- 记忆系统复杂度带来的隐私风险未充分披露
