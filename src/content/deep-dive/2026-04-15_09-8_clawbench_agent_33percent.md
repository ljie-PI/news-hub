# ClawBench：AI Agent 完成日常在线任务仅达 33.3% 成功率

**来源**: r/MachineLearning | 2026-04-14
**链接**: https://www.reddit.com/r/MachineLearning/comments/1slf7pg/

---

## 事件背景

机器学习研究者在 r/MachineLearning 分享了 ClawBench 基准测试，这是一个评估 AI 浏览器 Agent 在真实世界日常任务中表现的基准。与合成基准不同，ClawBench 使用 144 个真实上线的网站和 153 个任务，测试 Agent 在实际生产环境中的能力。结果发人深省：即使是最强模型 Claude Sonnet 4.6，成功率也仅有 33.3%。

## 核心观点/产品机制

**ClawBench 设计亮点**：
- 153 个任务，覆盖 144 个真实上线网站（非模拟环境）
- 测试类别：Finance（金融）、Academic（学术）、Travel（旅行）、Dev（开发）等
- 不同类别难度差异显著：Finance 和 Academic 类任务最高可达 50% 成功率；Travel 和 Dev 类最难，最佳模型也远低于 50%

**主要发现**：
- 最佳表现：Claude Sonnet 4.6，33.3% 成功率
- 第二名：GLM-5（智谱 AI），24.2%，令人意外的是这是一个**纯文本模型**（无视觉能力）
- 两者之间存在 9 个百分点的显著差距，远超第三名
- GLM-5 在纯文本场景（表单填写、文本搜索）超过了具有视觉能力的多模态模型

**为什么 33.3% 是个大问题**：
日常任务（订机票、查学术文献、填写表单）是 AI Agent 最"基础"的应用场景，连这些都只有 1/3 成功率，意味着现阶段 AI Agent 在无人监督的情况下无法被信任处理真实工作。

## 社区热议与争议点

- **GLM-5 的意外强势引发讨论**："一个中国纯文本模型排第二，这说明视觉能力在网页 Agent 中被高估了"（高赞评论）
- **"33% 意味着什么？"**：有研究者指出，33% 已经远好于随机基线，对于 153 个不同任务这是相当有意义的数字，但问题在于现实用户期望的是 90%+
- **对真实网站设计的抱怨**：部分讨论聚焦于，现实网站存在大量反爬虫机制、动态渲染、登录 wall 等，Agent 失败未必是模型问题
- **Travel 类为何最难？**：讨论认为旅行类任务（比价、多日期搜索、座位选择）涉及复杂的多步交互和动态内容加载，是当前 Agent 能力边界最明显的领域

## 行业影响与未来展望

ClawBench 的价值在于用真实数据打破了 AI Agent "已经能做大多数事情"的过度乐观叙事。这对以下领域有重要意义：

1. **AI Agent 创业公司**：需要更诚实地向客户说明当前能力边界，避免过度承诺
2. **浏览器自动化工具**（Perplexity Comet、GPT Atlas 等）：面临真实能力评估，需要找到可靠性的提升路径
3. **研究方向**：GLM-5 的表现提示，网页 Agent 的关键能力可能是"理解网页结构"而非"理解网页视觉"，这为轻量级模型路线提供了方向

距离 Agent 能可靠处理日常任务（>80% 成功率），行业还有相当长的路要走。
