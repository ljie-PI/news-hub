# 03 · microsoft/markitdown

> ⭐ 14,615 stars / week | 108,469 total | 2026-04-15

## a. 定位与痛点剖析

**MarkItDown** 是微软开源的文件转 Markdown 工具，专为 LLM pipeline 和 RAG（检索增强生成）场景设计。

核心痛点：企业和开发者手中存在大量格式多样的文档（PDF/PPT/Word/Excel/图片/音频），要将这些文档注入 LLM 上下文需要解决两个问题：
1. **格式转换**：不同格式需要不同解析库，集成成本高
2. **结构保留**：简单文本提取会丢失表格、列表、标题层级等结构信息，而这些对 LLM 理解至关重要

Markdown 成为中间格式的原因很清晰：主流 LLM 在训练时大量接触 Markdown 文本，天然"理解"其结构；同时 Markdown 接近纯文本，token 效率远高于 HTML/XML。

## b. 核心架构与技术细节

**技术栈**：Python 3.10+，模块化依赖设计

**支持的输入格式**：
- 文档类：PDF、DOCX（Word）、PPTX（PowerPoint）、XLSX/XLS（Excel）、EPUB
- 媒体类：图片（含 EXIF + OCR）、音频（EXIF + 语音转录）
- Web：HTML、YouTube URL
- 数据类：CSV、JSON、XML
- 压缩：ZIP（递归处理内容）

**架构设计**：
- `MarkItDown` 类是统一入口，内部通过 `DocumentConverter` 插件化架构分发处理
- 依赖按功能分组为可选包（`[pdf]`、`[docx]`、`[xlsx]`等），用户按需安装
- 支持第三方插件（`#markitdown-plugin` 标签在 GitHub 查找）
- 关键升级（0.1.0）：`convert_stream()` 统一使用二进制流，不再创建临时文件
- **MCP 集成**：`markitdown-mcp` 包提供 Model Context Protocol server，可直接接入 Claude Desktop

**高级功能**：
- Azure Document Intelligence 集成（`-d -e <endpoint>`）：企业级 OCR 和结构化识别
- LLM 图像描述（注入 `llm_client` 和 `llm_model`）：对图片和 PPTX 中的图像生成自然语言描述
- OCR 插件（`markitdown-ocr`）：通过 LLM Vision 从嵌入图像中提取文字，无需额外 ML 库

**基准对比**（来自 opendataloader 第三方测评）：
- markitdown 整体评分 0.589（12工具中第11名），表格准确率 0.273，标题准确率 0.000
- 这说明 markitdown 更擅长纯文字文档，复杂表格和标题层级识别较弱

markitdown 在精度上并非最优，但其优势在于：
1. **微软背书**（108k stars，企业可信度）
2. **生态完整**（MCP server、LangChain 集成、Azure 云服务对接）
3. **零门槛**（`pip install markitdown[all]`，三行代码即可使用）

## d. 开发者反馈与局限性

**正面**：
- 最受企业开发者信赖的 Microsoft 出品
- 108k total stars 说明广泛采用
- MCP 集成让其可以直接作为 Claude Desktop 的文档处理后端

**局限性**：
- 表格和标题识别准确率在同类产品中垫底，不适合处理结构复杂的财务报告、学术论文
- 对扫描版 PDF 支持有限（需要借助 Azure 付费服务或 OCR 插件）
- Python 3.10+ 限制了一些遗留系统的集成
- `markitdown-ocr` 插件需要配合 OpenAI API，增加了使用成本
