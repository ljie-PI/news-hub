## 一、定位与痛点

`Leonxlnx/taste-skill` 自称"The Anti-Slop Frontend Framework for AI Agents"，slogan 简单粗暴——**gives your AI good taste, stops boring slop**。痛点非常具体：Claude / Codex / Cursor / ChatGPT 默认生成的前端页面千篇一律——居中容器、灰底卡片、Tailwind 默认色、毫无层次的字体、零运动设计，俗称 "AI slop"。Taste-Skill 把"审美"封装成可移植的 **Agent Skills**，让代理在写 React/Vue/Svelte 前端时具备布局、排版、动效、留白方面的强约束，从源头压制套路化输出。仓库 18.4k stars、1.6k forks、MIT 协议，配套站点 `tasteskill.dev`。

## 二、架构与技术

它基于 Vercel Labs 的 `agent-skills` 规范，每个 skill 就是一份带 frontmatter 的 `SKILL.md`，通过 `npx skills add https://github.com/Leonxlnx/taste-skill` 一键拉取，并可用 `--skill design-taste-frontend` 单独安装。代码类技能包括默认全能型 **taste-skill (design-taste-frontend)**、GPT/Codex 严格变体 **gpt-tasteskill**、图→码的 **image-to-code-skill**、旧项目审计的 **redesign-skill**、柔和高端风 **soft-skill**、强制完整输出的 **output-skill**、Notion/Linear 风的 **minimalist-skill**、实验性瑞士排版 **brutalist-skill (BETA)**、Google Stitch 兼容的 **stitch-skill**。图像生成类则有 `imagegen-frontend-web`、`imagegen-frontend-mobile`、`brandkit` 三件套。`taste-skill` 文件顶部暴露 1–10 的**三档旋钮**：DESIGN_VARIANCE（布局实验度）、MOTION_INTENSITY（动效强度）、VISUAL_DENSITY（信息密度），用户可手动调档。

## 三、竞品与生态

同领域可对比的是 v0.dev、Magic Patterns、Galileo AI 这类"AI UI 生成器"，但它们是黑盒 SaaS；而 Taste-Skill 走另一条路——**把审美外挂注入到任意支持 Skills 协议的代理里**，框架无关，可与 Cursor、Claude Code、Codex、ChatGPT Images 等并用。它和最近兴起的 `awesome-claude-skills`、Anthropic 官方 Skills 体系处于同一生态层，差异化在于"垂直审美"和明确的反 slop 立场。仓库 `research/` 目录还公开了 root-causes / remediation / findings 三类研究材料，给出"为什么 AI 输出会变 slop"的理论依据。

## 四、反馈与局限

由于本质是 prompt 工程的集合，效果强依赖底层模型能力——同一份 SKILL.md 在 GPT-5 与 Claude Opus 上的表现差异显著，README 也因此专门拆出 `gpt-taste` 变体。安装名（`name:`）与文件夹名不一致，新手容易拷错。brutalist 仍是 BETA，输出稳定性偏低。另外作者特意声明"无任何官方代币"，可见项目走红后已遭遇蹭热度问题。
