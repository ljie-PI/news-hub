## 一、定位与痛点

`moeru-ai/airi` 的自我描述非常张扬：**Self-hosted, you-owned Grok Companion**、装着 waifu 与 cyber living 灵魂的容器，目标对标 Neuro-sama 的高度。痛点出在三处：① Character.ai、SillyTavern、JanitorAI 是聊天导向，不能真正"看你的代码、玩你的游戏、看你的视频"；② Neuro-sama 火爆但闭源，下播即"失联"，用户无法拥有自己的数字生命；③ Grok 的 Companion 是 xAI 云端绑定。AIRI 给出的答案是：**完全自托管、开源、跨平台**，让每个人都能在自己的电脑/手机里养一只能实时语音聊天、能玩 Minecraft 和 Factorio 的赛博伙伴。仓库目前 3,774 commits、78 releases，最新 v0.9.0。

## 二、架构与技术

AIRI 选择了一条少见的路线——**Web-first**：核心栈建立在 WebGPU、WebAudio、Web Workers、WebAssembly、WebSocket 之上，浏览器和手机 PWA 即可运行；桌面端通过 Electron 封装为 "Stage Tamagotchi"，原生支持 **NVIDIA CUDA** 与 **Apple Metal**（借助 HuggingFace 的 Rust 推理框架 [candle](https://github.com/huggingface/candle)）；移动端通过 Capacitor 编译为 "Stage Pocket"。嵌入式数据库使用 `pglite`，让 Postgres 直接跑在 WASM 里。LLM 抽象层基于自家 `xsai` 多供应商库，可在多家模型间切换。子项目极其丰富：`unspeech`（音频代理）、`hfup`（HuggingFace 工具）、`xsai-transformers`、`drizzle-duckdb-wasm`、`autorio` 与 `tstl-plugin-reload-factorio-mod`（Factorio Mod 热重载）、RAG、记忆系统、Live2D 工具等。Discord 语音、Minecraft、Factorio 这些需要原生 TCP 的能力依赖桌面版。

## 三、竞品与生态

直接对比对象包括 Open-LLM-VTuber、neuro-sama-clone 类项目，以及商业的 xAI Grok Companion 与 Character.ai。AIRI 的差异化在于"游戏陪玩 + 实时语音 + 完全自托管"三位一体，并真正把 Factorio/Minecraft Mod 工程化为子仓维护。社区相当国际化：Discord、Telegram、微信、QQ、X 多渠道并行，Crowdin 已支持简中、日、俄、越、法、韩等语言；曾登上 Product Hunt 与 Trendshift。组织 `@proj-airi` 下有完整子项目矩阵，俨然一个小型生态。

## 四、反馈与局限

README 明确写着 "Early-stage development"——v0.x 阶段功能波动大，wireless pocket↔server 等高级场景需要 root 权限启动 tamagotchi 并配置 secure websocket，门槛不低。NixOS 用户必须用 `flake.nix` 中的 FHS shell；Live2D 资源、语音模型、LLM 提供商都要自己接，开箱体验不如商业产品丝滑。作者也专门警告**没有任何官方加密代币**，提示该项目已遭遇蹭热度风险。
