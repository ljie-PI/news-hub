# terraink — 开源地图海报生成引擎

**仓库**: https://github.com/yousifamanuel/terraink  
**今日新增 Star**: +110  
**日期**: 2026-04-13

---

## 定位与痛点剖析

"城市地图艺术海报"是一个持续增长的细分市场——Etsy 上有无数出售定制城市地图挂画的店铺，售价通常在 $20-$80。传统上，这类海报要么需要高昂商业工具（Adobe Illustrator + 地图数据订阅），要么只能用有限的在线工具（MapPrint、MapBox Studio），要么需要 Python 环境 + 命令行的技术门槛（如 prettymaps）。

`terraink` 的定位是：**一个完全运行在浏览器中的免费开源地图海报生成器**，基于 OpenStreetMap 真实数据，支持高度自定义，导出 PNG/PDF/SVG 打印级质量。

作者 Yousuf Amanuel（来自德国汉诺威）在 Reddit 上明确表示，这是受 prettymaps 启发、但去掉所有技术门槛的版本：不需要克隆仓库、不需要 Python 环境、不需要命令行，打开网址就用。

---

## 核心架构与技术细节

### 全客户端渲染（100% Client-Side）

terraink 最重要的技术决策：**所有渲染逻辑都在浏览器端完成**，没有后端服务器处理地图数据。这意味着：
- 零运维成本（可部署到任何静态托管）
- 隐私友好（地图请求直接从用户浏览器发出）
- 可离线使用（在一定条件下）

### 数据源：Overpass API

地图数据来源于 **Overpass API**——OpenStreetMap 的查询接口。用户输入地名（如"Oslo"、"北京"），terraink 向 Overpass API 发送结构化查询，获取道路、建筑、水体、绿地等矢量数据，然后在浏览器中渲染成可定制的艺术海报。

### 七大 OSM 图层

terraink 支持 7 个可独立开关的 OSM 数据层：
- 道路网络（可按等级筛选）
- 建筑轮廓
- 水体（河流、湖泊）
- 绿地（公园、森林）
- 铁路
- 其他可定制图层

### 技术栈

- **框架**：React + Bun（Next.js 生态）
- **地图渲染**：基于 Canvas / SVG 的自定义渲染器（非依赖 MapBox/Leaflet）
- **数据**：Overpass API（OSM）
- **导出**：PNG、PDF、SVG 三种格式，适合打印级输出
- **许可证**：AGPL-3.0（强 copyleft）

### 自定义维度

用户可调整的参数包括：
- **颜色主题**：多种预设主题（暗色、亮色、单色、复古等）
- **布局**：20+ 种布局（不同纸张比例：A4、A3、正方形、Instagram、Twitter 等）
- **字体**：多种字体选择
- **图层开关**：每个 OSM 数据层独立控制
- **自定义标记**：可在地图上添加自定义位置标记
- **元素显隐**：地名标签、坐标信息、标题文字等

## 竞品对比与生态站位

| 工具 | 技术门槛 | 免费 | 开源 | 自定义度 | 输出质量 |
|------|---------|------|------|---------|---------|
| **terraink** | 无（纯 Web） | ✅ | ✅（AGPL） | 高 | 高 |
| prettymaps | Python + CLI | ✅ | ✅ | 极高 | 极高 |
| MapPrint | 无 | 部分免费 | ❌ | 中等 | 高 |
| Mapbox Studio | 中等 | 部分免费 | ❌ | 极高 | 高 |
| Etsy 定制卖家 | 无 | ❌（付费） | ❌ | 低 | 高 |

terraink 在"无门槛 + 高度定制"的交叉点上填补了空白。prettymaps 是技术用户的首选，但需要 Python 环境；商业工具要么收费要么功能有限；terraink 是目前最接近"完全免费 + 零门槛 + 功能完整"的选项。

作者也在 Reddit 评论中明确承认 prettymaps 的启发，并将 terraink 定位为"让更多人能用上同等质量工具"的普及版。

## 开发者反馈与局限性

**正面**：
- Reddit r/SideProject 社区高度正面：用户晒出了奥斯陆、罗马、特拉布宗等城市的精美效果
- Spatialists（地理信息专业媒体）推荐：称其为"customisation 广度突出"的工具
- 零门槛的承诺基本兑现

**局限**：
1. **Overpass API 限制**：免费 Overpass API 有速率限制，大城市数据量大时可能超时或失败
2. **浏览器性能**：复杂城市的大量矢量渲染对浏览器 CPU/内存有压力
3. **AGPL 许可证**：商业使用或二次开发需要开源代码，对商业项目有限制
4. **离线能力有限**：本质上依赖 Overpass API，无网络时无法获取新地图数据
5. **prettymaps 在定制深度上仍有优势**：可通过代码精确控制每个渲染参数，而 terraink 的 UI 有上限
