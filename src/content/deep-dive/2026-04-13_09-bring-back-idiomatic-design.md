## 事件背景

John Loeber 在其个人 Substack 发表了一篇名为《Bring Back Idiomatic Design》的文章，呼吁软件界回归"惯用设计"——即在整个平台/操作系统生态中保持一致的界面约定。文章以 Windows 2000/XP 时代的桌面软件为正面案例，批评现代 Web 应用界面设计的碎片化与混乱。文章在 HN 获得 377 票，引发 190 条讨论，触动了大量程序员对"那个年代设计好用"的集体记忆。

## 核心观点/产品机制

### 什么是"惯用设计"（Idiomatic Design）？

Loeber 定义"设计惯用语"（design idiom）为：一种如此普遍的界面模式，使用者无需思考即知如何使用，构建者也无需设计即会采用。经典例子：**复选框**。

"保持登录"这个问题，可以用文本输入框、下拉菜单、按钮来问，但任何网站都会用复选框——这就是惯用语。

### Windows 时代的设计优势

Loeber 赞美 Windows 2000 界面的几个特点：
1. **统一的菜单结构**：无论在 Photoshop 还是 Excel，File→Save, Edit→Redo, View→Fullscreen 都在同一位置
2. **键盘快捷键的一致性**：菜单项下划线字母指示快捷键，ALT+F 打开 File，N 打开新文件——可预期、可学习
3. **状态栏的信息密度**：页码、列号、字数、追踪修订状态一览无余
4. **文字优先的界面**：操作用文字标注，图标只在最显而易见时使用，"没有'我不知道这个按钮是干什么的'"

### 现代 Web 的设计混乱

作者举例：输入信用卡号有几十种不同的 UI 模式——文本框/下拉菜单/按钮各种变体，20 年过去了仍无共识。键盘快捷键因应用而异。"使用 Web 应用是无尽的'我想做什么，在哪里找到它？'循环。"

## 行业影响与未来展望

**为什么现在这篇文章引发共鸣？**

2026 年，AI 生成界面（AI-generated UI）开始出现——每次对话都能生成一个新的 UI。这在技术上令人兴奋，但从 Loeber 的视角来看，这是"惯用设计"的终极死亡：如果每个 AI 生成的应用界面都是独特的，用户将永远无法建立肌肉记忆，每次使用都是"重新学习"。

**几个值得关注的趋势：**
- **设计系统（Design System）的反击**：Tailwind UI、Radix UI、shadcn/ui 等组件库正在重建"惯用设计"——通过标准化组件，让不同应用的相似组件外观和行为一致。这是市场对碎片化的自发纠正。
- **AI UI 生成的双刃剑**：v0、Cursor 等 AI 工具能快速生成漂亮的 UI，但"漂亮"不等于"可用"。AI 目前缺乏对"为什么这个模式被所有人采用"的深层理解。
- **无障碍（Accessibility）要求将推动标准化**：随着欧盟 EAA（European Accessibility Act）2025 年生效，越来越多的 Web 应用将被要求使用语义化 HTML 和标准 ARIA 模式，这客观上会推动一定程度的惯用化。
- **移动优先设计对桌面的反向污染**：iOS/Android 设计语言（圆角、卡片、手势驱动）正在入侵桌面，但桌面用户的需求（键盘效率、信息密度）与移动端截然不同，这种设计移植往往以牺牲效率为代价。
