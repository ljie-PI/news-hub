# liteparse 深度解读

## 一、项目概览

**liteparse**（仓库：[run-llama/liteparse](https://github.com/run-llama/liteparse)）是 LlamaIndex 团队开源的一款轻量、快速的文档解析器，遵循 Apache 2.0 协议。与团队商业云产品 LlamaParse 形成互补：LlamaParse 专注于复杂排版、密集表格、扫描件、手写体等高难度场景，而 liteparse 则是一个**纯本地、零云依赖、不内嵌 LLM 特性**的 OSS 工具，主打"快"和"轻"。其核心能力是对 PDF 进行高质量的空间文本解析，输出带 bounding box 的文本结构。项目采用 Rust 实现，已发布 46 个 release，并提供统一的 `lit` CLI 入口。

## 二、核心特性

1. **多语言绑定**：原生 Rust 库，并通过 napi-rs、PyO3、wasm-bindgen 分别封装出 Node.js、Python、WASM 三种发行版，分别对应 `@llamaindex/liteparse`、`liteparse`、`@llamaindex/liteparse-wasm`。
2. **多格式输入**：通过 LibreOffice 自动转换 Office 全家桶（doc/docx/ppt/xlsx/odt/pages/key/numbers 等），通过 ImageMagick 处理常见图像格式，统一先转 PDF 再解析。
3. **离线 OCR**：默认捆绑 Tesseract，开箱即用；支持 `TESSDATA_PREFIX` 离线/内网部署；同时开放 HTTP OCR Server 规范（`OCR_API_SPEC.md`），可接入更高精度的远端 OCR 引擎。
4. **空间结构保留**：解析输出包含每段文本的坐标框，便于下游 RAG/Agent 重建版面与表格定位。
5. **Agent Skill 集成**：提供 `SKILL.md`，可直接挂载到支持 Skills 体系的 Agent 框架中。

## 三、CLI 与使用

`lit` 提供三个子命令：

- `lit parse <file>`：单文件解析，支持 `--format json|text`、`--target-pages "1-5,10"`、`--dpi`、`--password`、`--num-workers`、`--preserve-small-text` 等参数。
- `lit batch-parse <input-dir> <output-dir>`：批量解析，支持 `--recursive` 与 `--extension` 过滤。
- `lit screenshot <file>`：按页截图，方便配合视觉模型做混合解析。

OCR 默认并发数为 CPU 核数减一，`--max-pages` 默认 1000，DPI 默认 150，参数设计偏向"合理默认+按需调参"。

## 四、架构设计

仓库采用 Cargo workspace 组织：`crates/liteparse` 为核心库与 CLI，`pdfium` 与 `pdfium-sys` 封装 PDFium 的 FFI；`liteparse-napi/python/wasm` 三个 crate 分别承担跨语言绑定；`packages/` 下则是对应的 npm/PyPI 发布包，内含 TS/Python 薄封装并随包分发 native 二进制。这种"Rust 内核 + 多前端分发"的范式让单一实现可同时覆盖服务端、脚本、浏览器三类运行时，是当下 AI 基础设施工具的主流形态。

## 五、定位与价值

liteparse 的差异化在于"明确的边界感"：不与自家 LlamaParse 在复杂版面上正面竞争，而把"本地、快、可嵌入"做透——尤其 WASM 版本意味着可以在浏览器侧直接处理 PDF，避免敏感文档上云。对 RAG 工程而言，它是替代 PyPDF/pdfplumber 的强力候选，特别适合需要批量、低延迟、跨语言一致行为的生产链路。注意 V1 代码保留在 `logan/liteparse-v1` 分支，主线已进入新架构演进期。
