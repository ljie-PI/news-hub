# Google Search 推出"返回键劫持"垃圾内容新政策

> **来源**: Google Search Central Blog | **热度**: 904 pts, 511 评论 | **日期**: 2026-04-14

---

## 事件背景

Google 搜索团队宣布了一项新的垃圾内容政策，专门针对"**返回键劫持**"（Back Button Hijacking）行为。这一行为指网站通过技术手段干扰浏览器的历史记录导航——当用户点击"返回"按钮时，不是返回搜索结果页面，而是被重定向到另一个页面或停留在当前网站内。

这种手法在内容农场、低质量广告网站中极为常见，其目的是强制延长用户停留时间、增加页面浏览量以提升广告收入，同时阻止用户返回 Google 搜索其他结果。

---

## 核心观点/产品机制

**Google 的新政策要点：**
- 将返回键劫持行为明确列为**垃圾内容**（Spam），与 cloaking、doorway pages 等手法并列
- 具体包括：使用 `history.pushState()` 或 `history.replaceState()` 插入虚假历史记录条目、循环重定向、弹出拦截返回的对话框等
- 违规网站将面临搜索排名降级乃至被彻底移除搜索结果
- 政策将通过算法自动检测，同时支持人工审核举报

**技术背景**：返回键劫持本质上是对 HTML5 History API 的滥用。该 API 原本是为单页应用（SPA）实现流畅导航而设计的，但被恶意站长用于操纵浏览器历史记录。

## 社区热议与争议点

HN 511 条评论呈现出强烈的情绪反应，并涉及深层技术和政策讨论：

1. **"早该这么做了"的一致叫好**：绝大多数用户对这一政策表示欢迎，许多人分享了被返回键劫持折磨的亲身经历，认为这是严重破坏 Web 浏览体验的行为。一位用户写道："这种技术应该在 Web 标准层面就被禁止，而不是靠搜索引擎来管。"

2. **执法效果的质疑**：批评者指出 Google 历来声称会打击 SEO 垃圾，但内容农场依然横行，新政策能否真正落地存疑。讨论中有人总结："Google 和垃圾内容之间是一场永无止境的军备竞赛。"

3. **对合法 SPA 应用的影响**：前端开发者担心 Google 的检测算法是否会误伤使用了 `history.pushState()` 的单页应用。Google 文档中表示会区分"合理导航实现"和"蓄意劫持"，但边界如何划定引发争议。

4. **为何现在才管**：有评论追问为什么这一行为在业界已存在至少十年，现在才被明确列为违规。有人猜测可能与 Chrome 市场份额达到某个临界点，或内部搜索质量评分数据积累有关。

---

## 行业影响与未来展望

这一政策是 Google 持续加强"搜索结果质量"的一部分，近年来相关行动包括 Helpful Content Update、打击 AI 生成垃圾内容等。

对 Web 开发生态的影响：
- 合法 SPA 框架（React Router、Next.js 等）的历史记录管理方式可能需要进一步规范化
- 内容农场和低质量广告网站的生存空间将进一步压缩
- 浏览器厂商（如 Firefox、Safari）是否会跟进从浏览器层面限制此类行为，值得关注
