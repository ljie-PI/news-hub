# tinyhumansai/openhuman

## 定位与痛点剖析
OpenHuman 是一个开源、本地优先的个人 AI 助理系统，自我定位为"你的个人 AI 超级智能"：一个能记住你一切的大脑、一个编排智能体舰队的调度器、一个深度研究员。它要解决的痛点是"冷启动"——多数智能体上手时对你一无所知，往往要几天甚至几周喂上下文才真正有用。OpenHuman 主张几分钟内就把你了解透彻：连接账户后由自动抓取每二十分钟把数据拉到本地，再压缩成记忆，一次同步就让智能体掌握你的收件箱、日历、仓库、文档与聊天。它面向重视隐私、又想开箱即用图形界面的个人用户。README 称其发布一周内连续九天位居 GitHub 趋势榜首（自报口径），目前仍是早期 Beta。

## 核心架构与技术细节
项目核心用 Rust 编写，采用 GPL-3.0 协议。大脑部分的记忆树把数据压缩成评分过的 Markdown 树存进本地 SQLite，并镜像成可编辑的 Obsidian 知识库，作者称灵感来自 Karpathy 的 LLM 知识库，刻意避开"向量黑箱"。它宣称接入一百多个 OAuth、五千多个 MCP 服务器与九万多个技能，并用 TokenJuice 在工具输出进模型前压缩，号称最多省八成 token。编排层不是单循环而是基于开源 tinyagents 的检查点图，可暂停等审批、重启后续跑，卡住的子智能体会返回根因报告，它还内建浏览器、语音与会议智能体。

## 竞品对比与生态站位
README 用表格直接点名 Claude Cowork、OpenClaw 与 Hermes Agent 对比，强调自己在记忆、集成数量、可视化工作流、会议接入与本地隐私模式上的优势，并把后两者归为"终端优先、单循环、记忆靠插件"。它站位于"编排一切智能体的本地优先中枢"。

## 开发者反馈与局限性
issue 集中暴露性能问题：多个独立记忆检索串行执行，本该四十秒的操作耗时约一百四十一秒（#4680）；没有提示缓存使每轮重发二十一万余 token（#4679）；单次记忆检索即便有数据也要三四十秒（#4677）。作为早期 Beta，官方也直言"预期会有粗糙之处"。
