# 双 R9700 跑 Qwen3.6-27B + opencode：AMD 本地编程栈实战

## 背景

这是 r/LocalLLM 上一个非常硬核的本地配置分享：作者用 **两张 AMD Radeon AI Pro R9700**（基于 RDNA4 的工作站卡，单卡 32GB VRAM，2025 年下半年发布）跑 **Qwen3.6-27B**，并接入 **opencode**（一个开源的终端式编程 agent，类似 Claude Code / Aider 的 OSS 替代）。整个组合代表了 2026 年本地编程 agent 的一条"非 NVIDIA、非 Apple Silicon"的可行路径。

## 结构 B：技术栈拆解

1. **硬件层：双 R9700**
   - 总 VRAM 64GB，足以以 Q6/Q8 量化运行 27B 模型并留出充足 KV cache。
   - 相比 RTX 5090 / RTX Pro 6000，R9700 在每 GB 价格上更具优势，是预算敏感型本地用户的新选择。
   - 跨卡张量并行需依赖 ROCm 6.x + vLLM-ROCm 或 llama.cpp 的 Vulkan/HIP backend。

2. **推理层：很可能是 llama.cpp / vLLM-ROCm**
   - llama.cpp 对 AMD 的 Vulkan 后端在 2025 年成熟度大幅提升，部署门槛低于 ROCm。
   - 若追求 throughput，vLLM-ROCm + AWQ 量化是更优解，但配置复杂。

3. **模型层：Qwen3.6-27B**
   - Qwen3 系列在 2025 年成为开源编程模型的强力代表，27B 是甜区——比 7B 强很多，又比 72B 易部署。
   - 长上下文（128K+）与 tool-calling 原生支持，是适配 coding agent 的关键。

4. **Agent 层：opencode**
   - opencode 是开源的终端 AI 编程助手，支持任意 OpenAI 兼容 endpoint，因此能直连本地 llama-server / vLLM。
   - 提供文件读写、shell 执行、多步规划等能力，是 Claude Code 的 OSS 平替。

## 看点

- **完整闭环演示**：从硬件采购到 driver/ROCm 安装、模型下载量化、opencode 配置、实战使用——视频形式更适合让新人快速复制。
- **AMD 翻身信号**：长期以来本地 LLM 圈被 NVIDIA 垄断，R9700 + ROCm 6.x + 主流推理引擎的组合首次让 AMD 成为认真选项。
- **隐私 & 成本**：双 R9700 一次性投入约 $4-5k，但相比订阅 Claude Code Max（$200/月）+ API 用量，1-2 年即可回本，且代码不外流。

## 价值评估

这类帖子在 r/LocalLLM 价值极高：它不是 benchmark 截图，而是 **可复现的端到端 setup 教程**。对想搭建本地编程工作站的开发者，这是非常实用的参考。关注点：(1) ROCm 在该卡上的稳定性；(2) 27B 在真实长 codebase 任务上的表现是否接近 Claude Sonnet；(3) opencode 的 agent loop 成熟度。AMD + 开源模型 + 开源 agent 的"全开源工作流"正在成为现实。
