# Yansu：学习你的工作方式，并把它"软件化"

## 一句话定位

**AI that learns how you work and turns it into software** —— 观察用户的实际工作流程，然后将这些重复模式自动转化为可执行软件 / agent。

## 背景：从 "Workflow Automation" 到 "Workflow Synthesis"

n8n、Zapier、Make 这类传统自动化工具要求用户**先设计 workflow**；而 2025-2026 年兴起的新范式则反过来：让 AI **观察用户实际操作**（屏幕录制、浏览器事件、键鼠日志、文档编辑），自动归纳出可复用流程并生成代码或 agent。代表案例包括 Adept ACT-1、Anthropic Computer Use、OpenAI Operator、以及大量 "AI RPA" 创业公司。Yansu 属于这一阵营，但更强调 **"turn into software"** —— 输出形态不只是一次性执行，而是持久化的软件资产。

## 结构 B：可能的产品架构

1. **观测层（Telemetry）**：通过桌面 agent / 浏览器扩展 / IDE 插件捕获用户操作轨迹（含上下文：打开的文件、点击的按钮、输入的内容）。
2. **归纳层（Pattern Mining）**：用 LLM 分析行为流，识别重复子任务（"每周一从 Stripe 拉数据填入 Sheet 再发邮件"）。
3. **合成层（Code/Agent Gen）**：将识别出的模式转化为脚本、API workflow、或可调用 agent。可能产出 Python script、TypeScript 函数、或符合 MCP 协议的 tool。
4. **执行 & 反馈层**：用户运行生成的软件后，可继续记录纠错，模型迭代优化。

## 差异化

- 相对 Zapier / n8n：免设计，AI 主动发现自动化机会。
- 相对纯 computer-use agent：不是每次都"看屏幕重做一遍"，而是把行为固化成 deterministic 软件，效率与可靠性更高。
- 相对 Cursor / Copilot：聚焦"非开发者"用户的日常工作流自动化，受众更广。

## 风险与挑战

- **隐私敏感性**：屏幕录制级别的观测在企业环境会遇到强阻力，需要本地化 / 端侧推理保障。
- **冷启动**：观察需要时间，用户在看到价值之前可能流失。
- **合成准确性**：从行为反推业务意图是难点，错误归纳会生成无用甚至有害的软件。

## 价值评估

Yansu 触及的是一个非常大的命题——**"如何让 AI 替每个人写定制软件"**。如果能在隐私、准确性、易用性三角中找到平衡，将极具想象空间。但同时也是难度极高的赛道，需要在数据飞轮和企业信任之间长期投入。PH 首发是关键一刻，值得关注其 demo 质量与早期用户反馈。
