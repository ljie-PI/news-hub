## 产品概览

Powabase 是当日 Product Hunt 排名第 2 的开发者工具，定位是\"**AI 原生应用的一站式后端**\"。它把 Postgres、对象存储、Auth、Realtime、**RAG**、**Agent 编排**、**可视化工作流**整合在同一平台，专门面向\"用 AI 编码 Agent 构建 AI 应用\"的新型开发者。官方主张：**up to 70% token 节省、2-4× 更低构建成本、1 周从零到 MVP**。

## 核心能力

### 1. BaaS 基础层
- **Postgres + RLS**：与 Supabase 类似的关系型 + 行级安全模型；
- **Auth / 对象存储 / Realtime**：开箱即用的认证、文件、实时同步；
- **PostgREST + 直连**：可通过 REST/GraphQL 风格 API 或数据库直连访问。

### 2. RAG 开箱即用
- 上传 PDF、图片、Office 文档、URL，自动 **抽取-切块-嵌入-索引**；
- **OCR 准确率 91%**（OlmOCR-Bench），**RAG pipeline 准确率 98.7%**（FinanceBench）；
- 支持 **BM25 + pgvector + 混合检索 + SOTA reranker**；
- 处理多模态内容索引。

### 3. Agent 与工具
- 定义 **ReAct 编排**：可组合多个 LLM、知识库、工具；
- 运行通过 **SSE 流式输出**，完整记录检索事件、工具调用、token delta、引用；
- 内置 web search、code execute，并允许通过 **HTTP 或 MCP** 接入自定义工具；
- Session 管理多轮状态。

### 4. 可视化工作流
- 拖拽 trigger、condition、agent、HTTP、code 等节点；
- 自然语言 copilot 可帮你生成流程；
- 工作流可一键发布为 HTTP 端点。

### 5. 架构与部署
- **专用计算**：每个项目独立栈、独立资源、独立备份，无邻居噪音；
- **三种部署模式**：Powabase Cloud / Docker / Kubernetes（含 Helm）；
- **BYO Keys**：支持 OpenAI、Anthropic、Google、OpenRouter，密钥按项目加密存储。

### 6. \"为 Agent 而生\"
- 文档结构化优化，专门给编码 Agent 读；
- 平台原生说 MCP，**Claude Code、Codex、Cursor 可直接在 Powabase 上构建** 无需人工指引。

## 与 Supabase / Firebase 的关键差异

虽然名字与 Supabase 接近，但 Powabase 把**\"AI 原生\"作为一等公民**而非可选插件：
- Supabase 的 vector 是 extension；Powabase 的 RAG 是产品核心，包含 OCR、reranker、引用追溯；
- Firebase 没有 Agent / MCP 体系；
- Powabase 内置工作流引擎、Agent 运行时与可观测性 dashboard，是\"BaaS + LangGraph + Pinecone\" 的合体形态。

## 客户与口碑

官网披露的客户包括 **MIT、Cegid、Ytel、Evergreen、Ryden、OACIO、DAOU、Judocu**——既有学术机构也有 B 端 SaaS，说明产品已经过真实场景验证。

## 适用场景

1. **企业内部 AI 应用**：HR 知识库问答、合同审阅、运营自动化；
2. **AI 代理产品**：以 Agent 为核心交互的 SaaS，需要稳定的工具调用 + 多轮状态；
3. **代理商/咨询公司**：用 1 周 MVP 速度交付客户定制 AI 项目；
4. **vibe coding 团队**：让 Claude Code/Cursor 直接搭建后端，避免在底层胶水代码上耗费精力。

## 启示

- **\"AI 原生 BaaS\"** 正在成为独立赛道。Supabase 路线被\"AI 加 RAG 加 Agent\"路线超车的可能正在加大；
- **MCP 已是 Backend 平台的入场券**，没有它就接不上主流 Agent 工具；
- **可观测性** 被列为一等功能：token、工具调用、检索事件都需要可见，否则生产级 Agent 应用根本调不动；
- **BYO Keys + 自托管** 同时存在，是企业市场必须的妥协。

## 总结

Powabase 押注的是\"AI 应用的下一代 Supabase\"。它把数据、检索、Agent、工作流、可观测性塞进一个统一抽象，并以 MCP 为协议层，与 Claude Code/Cursor 等编码 Agent 天然契合。对希望快速搭出生产级 AI 应用的团队，这是值得纳入候选名单的全栈方案。
