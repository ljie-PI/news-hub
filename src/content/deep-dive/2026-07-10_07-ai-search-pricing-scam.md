# The pricing for AI search is such a scam...

## 1. 事件背景

在 r/AI_Agents 社区，用户 Sleek65 发布了一篇火药味十足的帖子，直指当下 AI 搜索服务的定价"就是一场骗局"。他指出，市面上几乎所有面向 AI Agent 的搜索 API 提供商——例如 Exa、Tavily 等——收费区间普遍在每 1000 次搜索 5 到 25 美元之间。对于需要为 Agent 大量调用网页检索的开发者而言，这笔开销会随调用量迅速膨胀，成为成本结构中难以忽视的一块。

## 2. 核心观点

Sleek65 的核心论点是：这些服务的定价与其实际提供的技术价值严重不匹配。为了证明这一点，他自己动手搭建了一套内部搜索工具，声称在成本不到 3 美元的情况下完成了 10 万次以上的网页搜索，且检索结果质量与那些昂贵的商用 API 相当。更进一步，他将这套工具开源并免费提供，意在直接戳破"AI 搜索必须很贵"这一行业叙事，让开发者拥有低成本的替代选择。

## 3. 社区热议与争议点

帖子在社区引发了明显分歧：

- **支持方（Pros）**：不少开发者对高昂的搜索 API 账单深有共鸣，认为商用提供商的溢价主要来自品牌与便利性，而非不可替代的技术壁垒；开源方案让小团队得以摆脱按次计费的束缚。
- **质疑方（Cons）**：也有人指出，自建工具隐藏了服务器、代理 IP、反爬维护与法律合规等成本，3 美元的账面数字并未完整反映真实投入。
- **质量之争**：部分评论者质疑"结果相当"的说法缺乏严谨基准测试，商用 API 在排序、去重与时效性上的工程沉淀难以被简单复制。
- **规模化担忧**：还有声音提醒，自建方案在生产环境下的可靠性，往往才是企业愿意付费的原因。

## 4. 行业影响

这场讨论折射出 AI 搜索赛道的定价合理性拷问。随着开源检索方案增多，商用提供商可能被迫以更透明的成本结构、更强的企业级 SLA 来证明溢价。对开发者来说，"买还是自建"的权衡将愈发常见，成本敏感场景或率先转向开源替代品。
