# 04 · multica-ai/multica

> ⭐ 9,823 stars / week | 12,474 total | 2026-04-15

## a. 定位与痛点剖析

**Multica** 定位为**开源的托管式 AI Agent 协作平台**，口号是"你接下来的10个招聘不会是人类"。

它解决的核心痛点是：当前 AI Coding Agent（Claude Code、Codex等）使用门槛高、协作体验差的问题：
1. **无团队感**：Agent 是工具而非协作者，无法像同事一样被分配任务、汇报进度、参与对话
2. **无状态持久化**：每次调用 Claude Code 都是从零开始，没有"这个 Agent 已经熟悉这个项目"的概念
3. **技能不可复用**：团队中每个人都重复摸索相同的 Agent 技巧，没有技能沉淀机制
4. **多工具割裂**：Claude Code、Codex、OpenClaw、OpenCode 各自独立，无统一管理界面

Multica 的解法是将 Agent 作为"团队成员"而非工具来建模：Agent 有个人主页、出现在任务板上、能主动汇报阻塞问题、技能可被整个团队复用。

## b. 核心架构与技术细节

**技术栈**：
- 前端：Next.js 16（App Router）
- 后端：Go（Chi router + gorilla/websocket + sqlc）
- 数据库：PostgreSQL 17（含 pgvector 向量扩展）
- Agent 运行时：本地 Daemon（自动检测 claude/codex/openclaw/opencode CLI）

**核心架构**：
```
Next.js 前端 ↔ Go 后端(Chi+WS) ↔ PostgreSQL(pgvector)
                    ↕
              Agent Daemon（本地机器）
         [Claude Code/Codex/OpenClaw/OpenCode]
```

**关键机制**：
1. **Runtime 概念**：用户本地机器通过 `multica setup` 注册为 Runtime，后台 Daemon 实时上报可用 CLI 工具
2. **任务生命周期**：enqueue → claim → start → complete/fail，WebSocket 实时推送进度
3. **技能复用**：团队内所有 Agent 的成功解决方案自动抽象为 Skill，供后续任务复用
4. **Multi-Workspace**：工作空间级隔离，每个工作空间有独立的 Agents、Issues 和配置

**安装**：
```bash
brew install multica-ai/tap/multica  # macOS
multica setup  # 连接 Cloud，启动 Daemon
```

## c. 竞品对比与生态站位

| 维度 | Multica | Paperclip（同类） | Linear+AI | GitHub Issues+Agent |
|------|---------|-----------------|-----------|-------------------|
| 定位 | 团队AI协作平台 | 单人AI公司模拟器 | 项目管理+AI辅助 | PR自动化 |
| 用户模型 | 多用户+角色权限 | 单操作员 | 多用户 | 开发者 |
| Agent交互 | Issues+对话 | Issues+心跳 | 建议式 | 自动PR |
| 部署 | 云优先/自托管 | 本地优先 | SaaS | SaaS/自托管 |
| 技能系统 | ✅ | ✅(+插件) | ❌ | ❌ |

Multica 与 Paperclip 的定位差异很清晰：前者服务于**真实的多人开发团队**，后者是单人用户玩 AI 公司角色扮演。随着 AI Coding Agent 进入团队日常，Multica 押注的方向有真实市场需求。

## d. 开发者反馈与局限性

**正面**：
- 12k total stars（较新项目）增速健康
- 与 Claude Code/Codex/OpenClaw/OpenCode 全面兼容，降低迁移阻力
- 技术栈选择务实（Go 后端性能好，pgvector 为未来向量检索铺路）

**局限性**：
- 强依赖 Multica Cloud（虽支持自托管，但自托管需 Docker，配置成本不低）
- 团队协作 AI 市场还在早期，教育成本高（说服团队采用 AI 队友比采用工具难得多）
- pgvector 的向量能力目前在产品中体现不明显，可能是规划中的功能
- 与 GitHub/GitLab Issue 系统没有深度集成，使用独立看板意味着工作流割裂
