# zml/zml

## 定位与痛点剖析
ZML 是一套面向生产的 AI 推理栈，核心诉求是把 AI 工作负载与专有硬件解耦。当前主流推理框架（vLLM、Ollama 等）多以 Python 包裹 CUDA 内核，深度绑定 NVIDIA 生态，导致跨硬件迁移困难、依赖链笨重。ZML 目标是「任意模型、任意硬件、零妥协」，让同一份代码在多种加速器上获得峰值性能。

## 核心架构与技术细节
ZML 几乎完全用 Zig 语言编写（自报口径约占代码 92.7%），结合 MLIR 与 OpenXLA，将模型计算图直接编译为独立原生二进制，可面向 NVIDIA、AMD、Intel、Google TPU、AWS Trainium 编译。构建使用 Bazel，能在构建期按需下载并打包 CUDA、ROCm 等运行时的最小依赖，显著减小 OCI 镜像或 tar 产物体积。示例含 Llama、Qwen、LFM 的统一 LLM CLI。

## 竞品对比与生态站位
相较 vLLM、Ollama 依赖 Python 与 PyTorch，ZML 走的是编译式、无 Python 依赖的路线，主打硬件无关性与部署可移植性，能把模型直接产出为单一原生二进制并封装为容器镜像。它站位于底层 AI 工程与多加速器生产部署，尤其契合 AMD ROCm 生态逐步成熟、消费级 GPU 本地推理日益可行的趋势，是「值得关注」的一类基础设施项目。

## 开发者反馈与局限性
项目 star 约 3.8k、fork 约 167（自报口径），并在 2026 年 3 月发布 v2 版本，拥有活跃的贡献者社区。主要局限在于 Zig 语言与 Bazel 构建系统的学习曲线相当陡峭，对习惯 Python 生态的算法工程师门槛较高；同时其模型覆盖、算子支持与工具链完善度仍不及成熟框架，整体社区规模偏小，生产落地案例有限。
