# louis-e/arnis

## 定位与痛点剖析
Arnis 面向 Minecraft 玩家与地理数据爱好者，解决"想把真实世界的城市与地貌高精度还原进 Minecraft，却缺乏自动化工具、只能手动搭建"的核心痛点。它能够根据现实的地理、地形与建筑数据，生成高细节的 Minecraft Java 版（1.17+）与基岩版世界，让用户轻松再现自己的家乡、大型城市乃至山川湖泊等自然景观，把原本极其耗时的巨型建造工作交由算法自动完成，大幅降低还原真实世界的时间与技术成本。

## 核心架构与技术细节
项目以 Rust 语言编写，核心算法负责处理来自 OpenStreetMap 的地理空间数据，并结合高程数据一并计算，生成能够还原真实地形与建筑轮廓的 Minecraft 世界。它借助 Overpass API 拉取 OSM 数据，采用 Tauri 构建跨平台桌面客户端，能够处理大规模地理数据集，技术栈覆盖 maps、openstreetmap、osm 等主题，整体采用 Apache 2.0 开源许可协议，免费且可自由使用。

## 竞品对比与生态站位
相较于部分需要付费或强制在线的地图转换工具，Arnis 完全免费开源、可在本地独立运行，并直接对接开放的 OSM 数据生态；官方另行提供浏览器端的 MapSmith 来满足移动端生成与更大地图尺寸的需求，从而形成本地客户端与云端服务互补的生态站位。

## 开发者反馈与局限性
自报口径约 17k stars，社区人气相当旺盛。其主要局限在于生成质量高度依赖 OSM 数据的完整度，地图数据缺失或错误会直接影响还原精度；同时大范围地图生成对本地内存与算力要求较高，超大尺寸地图往往仍需转向在线方案。
