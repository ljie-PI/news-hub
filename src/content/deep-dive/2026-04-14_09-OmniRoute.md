# OmniRoute — AI 多供应商 LLM 网关

## 1. 定位与痛点剖析

OmniRoute 是一个**免费开源的 AI 网关**，定位为"多供应商 LLM 统一代理"。核心诉求是解决开发者在使用 AI 编程工具时面临的三大痛点：

- **订阅费用高且额度经常用尽**：Claude Pro、Codex Pro 每月 $20-200，但用户经常在写代码中途被限流中断，破坏专注状态。
- **多供应商 API 格式不统一**：OpenAI、Claude、Gemini 各有不同格式，切换需重新配置 SDK 和 endpoint。
- **地区封锁**：OpenAI 等服务对部分国家地区封锁，OAuth 流程失败。

OmniRoute 的解法是：对所有工具（Claude Code、Codex、Gemini CLI、Cursor、Cline 等）暴露统一的 `http://localhost:20128/v1` 端点，内部实现智能路由、格式翻译和自动降级。

## 2. 核心架构与技术细节

**技术栈**：100% TypeScript，基于 Next.js 构建，数据库使用 SQLite（better-sqlite3），支持 npm 全局安装和 Docker 多平台（AMD64 + ARM64）部署，以及 Electron 桌面应用。

**四层降级路由机制（核心设计）**：
```
Tier 1: 订阅制账号（Claude Code、Codex、Gemini CLI）
  → 额度耗尽
Tier 2: API Key（DeepSeek、Groq、xAI、Mistral 等）
  → 预算超限
Tier 3: 低价模型（GLM $0.6/1M、MiniMax $0.2/1M）
  → 预算超限
Tier 4: 免费模型（Qoder、Qwen、Kiro — 无限量）
```

**格式翻译层**：自动处理 OpenAI ↔ Claude ↔ Gemini 之间的协议转换，包括 `<think>` 标签提取（DeepSeek R1）、角色规范化（`developer` → `system`）、Gemini structured output 自动转换等。

**弹性机制**：Circuit Breaker（每模型独立，Closed/Open/Half-Open 状态）、指数退避、反雷暴保护（Mutex + Semaphore）、语义缓存（二级缓存）、请求幂等性（5秒去重）。

**协议支持**：除 OpenAI 兼容代理外，还内置 MCP Server（25 个工具）和 A2A Protocol（Agent-to-Agent），支持任务生命周期管理、SSE 流式传输、Agent Card 自动发现。

**安全**：AES-256-GCM 加密凭证、API Key 模型级权限控制、CSRF 保护、IP 过滤、Prompt Injection Guard。

| 项目 | 定位 | 免费模型支持 | 本地部署 | MCP/A2A |
|------|------|-------------|---------|---------|
| **OmniRoute** | 免费优先的全功能网关 | ✅ 原生支持 | ✅ npm/Docker | ✅ |
| LiteLLM | 企业级 LLM 代理 | 部分 | ✅ | ❌ |

OmniRoute 的差异化在于：**免费供应商原生集成**（Kiro、Qoder、Qwen 等）、**Electron 桌面应用**、**30 语言 i18n**、**MCP+A2A 双协议**，以及**CLI 指纹模拟**（降低账号风控风险）。

## 4. 开发者反馈与局限性

**优势**：
- 对零成本编程场景覆盖极佳，`Free Stack` 组合可实现真正的 $0/月 AI 编程
- 文档极详尽，覆盖 30 个具体痛点场景和实操 Playbook
- 国际化支持好（30 种语言），对非英语用户友好

**局限性**：
- 依赖第三方免费服务的稳定性（Qoder、Kiro 等服务随时可能变更）
- 部分"免费"供应商使用了非官方 OAuth 流程，存在账号封禁风险
- 单人维护项目（作者 diegosouzapw），长期维护稳定性存疑
- 新增供应商时，格式转换的维护成本较高
