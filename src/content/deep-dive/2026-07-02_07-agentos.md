# rivet-dev/agentos

## 定位与痛点剖析
agentOS 是 rivet-dev 开源的、面向 AI Agent 的可移植"操作系统"，定位为比沙箱更快、更轻、更省的替代方案。它解决的痛点是：传统沙箱要启动 VM 或拉起容器，冷启动慢、内存占用大、与业务后端集成复杂。agentOS 主打在你自己的进程内运行 Agent，毫秒级启动、可直接调用后端函数。面向要把编码 Agent 嵌入自家后端的开发者。

## 核心架构与技术细节
核心用 Rust 编写，以 npm 包形式交付，在进程内跑一个轻量 VM，采用"浏览器同款"隔离技术，对文件、网络、进程默认拒绝、按需授权。内置 Pi、Claude Code、OpenCode 三种 Agent，会话经 ACP（Agent Communication Protocol）创建与恢复，并提供统一转录格式和自动持久化。支持挂载 S3、Google Drive 等外部存储为文件系统，还有 Bindings、Cron、Webhooks 等编排能力。Apache 2.0 许可。

## 竞品对比与生态站位
直接对标 E2B、Daytona 等沙箱服务。官方自报冷启动约 6ms、比最快沙箱快数十至数百倍、成本最多省 32 倍（均为自报口径，基准为 2026 年 3 月自测）。与 CubeSandbox 相比，二者都用 Rust，但 agentOS 强调进程内嵌入与 Agent 编排，而非独立沙箱服务；官方称二者可通过 sandbox 扩展互补。

## 开发者反馈与局限性
进程内运行是双刃剑：隔离强度理论上弱于独立内核的 VM，重负载仍需挂载完整沙箱来补足。项目较新（star 约 3400），生态与第三方验证有限。所有性能数字均来自官方基准，缺乏独立复现；对 Rivet Cloud 商业生态的潜在绑定也需留意。
