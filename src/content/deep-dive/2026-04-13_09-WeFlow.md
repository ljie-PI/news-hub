# WeFlow — 本地微信聊天记录导出与分析工具

**仓库**: https://github.com/hicccc77/WeFlow

## 定位与痛点剖析

微信是中国最主要的即时通讯工具，但它的数据管控极为封闭：
- 聊天记录无法原生导出（官方只支持迁移，不支持导出）
- 历史消息被撤回后用户无法保留
- 无法对聊天记录做统计分析
- 朋友圈内容有时间访问限制（只能看最近半年）

`WeFlow` 解决的是**微信数据主权**问题——让用户能够访问、分析、导出自己的聊天记录，完全在本地运行，不上传任何数据到云端。

产品定位横跨三个需求层次：
1. **实用**：防撤回、消息导出、联系人管理
2. **分析**：年度报告、群聊画像、关系分析
3. **开发**：HTTP API 供二次开发集成

## 核心架构与技术细节

### 技术实现原理

WeFlow 的核心是**微信本地数据库的实时读取和解密**。微信在本地以加密数据库存储聊天记录（SQLite + 自定义加密），WeFlow 通过内存注入或密钥提取技术获取解密密钥，实时读取聊天记录而**无需生成解密中间数据库**。

这一点是 README 特别强调的技术优势：区别于传统的"先解密整个数据库"方案，WeFlow 实现真正的实时读取。

**仅支持微信 4.0 及以上版本**（新的数据库格式）。

### 功能矩阵

| 模块 | 详细功能 |
|------|---------|
| **聊天查看** | 解密图片/视频/实况（Google 协议拍摄）；支持修改/删除本地消息；实时刷新 |
| **防撤回** | 拦截他人撤回操作，保留原始消息 |
| **通知** | 新消息桌面弹窗，支持黑白名单 |
| **私聊分析** | 消息数量统计、类型分布、时段分析 |
| **群聊分析** | 成员信息、发言排行、活跃时段、媒体统计 |
| **年度报告** | 按年统计或跨年历史报告 |
| **双人报告** | 选定好友，生成专属关系分析 |
| **消息导出** | JSON、HTML、TXT、Excel、CSV、PGSQL、ChatLab 格式 |
| **朋友圈** | 解密内容、导出、拦截删除/隐藏、突破时间限制 |
| **联系人** | 导出好友/群组/公众号；尝试找回删除好友 |
| **HTTP API** | 本地 5031 端口，供二次开发集成 |

### 技术栈

- **桌面框架**：Electron（Node.js）
- **开发命令**：`npm install && npm run dev`
- **平台支持**：Windows（x64）、macOS（Apple Silicon）、Linux（x64，含 AppImage）
- **ArchLinux**：`yay -S weflow` 直接安装
- **HTTP API 端口**：127.0.0.1:5031，原始 JSON 或 ChatLab 格式

### HTTP API 集成示例

WeFlow 提供本地 HTTP API，可与其他工具集成：
- 启用路径：设置 → API 服务 → 启动服务
- 支持格式：原始 JSON 或 ChatLab 专属格式（供 [ChatLab](https://chatlab.fun/) 深度分析）
- API 仍处于早期阶段，接口可能变动

## 开发者反馈与局限性

**正面**：
- 功能全面，解决了微信数据导出的长期痛点
- 实时读取（无需中间数据库）的技术方案被用户高度评价
- 多平台支持（含 Linux）在同类工具中罕见

**局限**：
1. **灰色地带**：微信官方服务协议禁止逆向工程。WeFlow 本身强调"请负责任地使用，遵守相关法律法规"，但本质上依赖非官方技术手段
2. **版本依赖**：仅支持微信 4.0+，旧版本用户无法使用；微信更新可能导致功能失效
3. **Windows 限制**：仅支持 x64，不支持 ARM Windows 设备
4. **Linux 实况解密**：实况（Live Photo）解密仅支持谷歌协议拍摄，局限性存在
5. **HTTP API 早期阶段**：接口不稳定，不适合生产环境依赖
6. **找回好友功能不完善**：README 明确标注"尚不完善"
