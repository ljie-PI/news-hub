## 定位与痛点剖析

Claude Code 是 Anthropic 推出的 AI 编程助手，但它有一个根本性缺陷：**每次会话结束，上下文全部清空**。开发者在上一个 session 中让 Claude 搞懂的架构、排查的 bug、建立的约定，下一次全都要重新解释。这不仅浪费时间，也让"AI 结对编程"的体验大打折扣——你需要永远充当 Claude 的记忆。

`claude-mem` 精准解决这个痛点：它是一个 **Claude Code 插件**，在后台自动捕获每次编程会话中 Claude 执行的所有工具调用，用 AI（Claude Agent SDK）将这些原始输出压缩成约 500 token 的精炼观察，并在下次会话开始时自动注入相关历史上下文。

这是一种"**无感知记忆增强**"——开发者不需要手动维护任何文档或提示文件，claude-mem 在幕后完成整个记忆管道。

## 核心架构与技术细节

claude-mem 的架构分为三层：

### 1. Hook 层（实时捕获）
通过 Claude Code 的 `PreToolUse` 和 `PostToolUse` 钩子，插件拦截每一次工具调用（文件读写、shell 命令、搜索等），将工具名称、输入、输出全部记录下来。这是实时、透明的，不影响 Claude 的正常执行流程。

### 2. 压缩层（Worker 服务）
捕获的原始工具输出被送入一个后台 Worker 进程，使用 **Claude Agent SDK** 将其压缩为约 500 token 的 "observation"（观察）。压缩是语义级别的——保留关键发现、省略冗余细节。每个压缩观察保留完整的元信息（时间戳、工具类型等）便于后续检索。

**"Endless Mode"**：这是 claude-mem 的核心创新。它将工作记忆（Working Memory）和归档记忆（Archive Memory）分开：
- **Working Memory（上下文内）**：压缩后的观察，约 500 tokens/条
- **Archive Memory（磁盘）**：完整原始工具输出，永久保留用于审计

### 3. 注入层（会话启动）
新 session 开始时，插件基于相关性检索历史观察，自动拼接进系统提示或上下文头部，让新的 Claude 实例"记得"之前做过什么。

**技术栈**：
- Node.js / TypeScript
- Claude Agent SDK（压缩调用）
- Claude Code Plugin API（`PreToolUse`/`PostToolUse` hooks）
- 本地文件系统（观察存储）
- npm 包 `claude-mem`（但注意：`npm install -g claude-mem` 只安装 SDK，**不注册 plugin hooks**，需按文档单独注册）

**安装方式**（正确路径）：
```bash
# 通过 Claude Code 插件系统安装（非 npm 全局安装）
/plugin add thedotmack/claude-mem
```

## 竞品对比与生态站位

| 方案 | 方式 | 自动化程度 | 痛点 |
|------|------|-----------|------|
| **claude-mem** | Plugin hook + AI压缩 | 全自动 | 依赖 Claude Code |
| **手动 CLAUDE.md** | 开发者手写 | 全手动 | 需要持续维护 |
| **mem0** | 独立记忆层服务 | 半自动（需集成） | 需要外部 API/服务 |
| **Cursor Rules** | 静态规则文件 | 无 | 不记录动态会话内容 |
| **MemGPT/Letta** | Agent 记忆框架 | 需要完整 Agent 系统 | 重量级，非 Claude Code 原生 |

claude-mem 的核心优势是**零配置、原生集成**——它寄生在 Claude Code 的插件系统内，不需要开发者改变任何工作习惯。在 Claude Code 生态日益壮大的背景下，这个位置非常有价值。

劣势在于强依赖性：仅支持 Claude Code，不能用于其他 AI 编程工具（Cursor、GitHub Copilot 等）。

## 开发者反馈与局限性

凭借 814 stars/天的爆发，claude-mem 显然踩中了真实痛点。从社区反馈来看：

**正面评价**：
- "终于不用每次重新解释项目架构了"
- 压缩后的 500-token 观察被认为信噪比合理
- 安装简单，通过 Claude Code 插件市场一键安装

**已知局限**：
1. **仅适用 Claude Code**：不支持 Cursor、Windsurf、GitHub Copilot 等
2. **npm 安装陷阱**：`npm install -g claude-mem` 安装的是 SDK 库，**不会注册 plugin hooks 或启动 worker 服务**，README 特别强调这一点
3. **压缩质量依赖 Claude**：若 Claude 压缩时遗漏重要细节，下次会话的上下文注入可能不准确
4. **存储无上限管理**：长期使用后观察文件会积累，目前未见自动清理机制
5. **隐私风险**：所有工具输出被发送给 Claude API 进行压缩，敏感代码/凭证有泄露风险
