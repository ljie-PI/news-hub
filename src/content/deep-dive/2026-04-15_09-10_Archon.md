# 10 · coleam00/Archon

> ⭐ 4,024 stars / week | 17,962 total | 2026-04-15

## a. 定位与痛点剖析

**Archon** 是"AI Coding 的第一个开源 Harness Builder"，核心价值是让 AI Coding 工作流**确定性且可重复**。

类比很精准：Docker 让基础设施可重复，GitHub Actions 让 CI/CD 可重复，Archon 让 AI Coding 流程可重复。

核心痛点：当你让 AI Agent 执行"修复这个 bug"时，每次执行结果都不同——有时跳过测试，有时忘记写 PR，有时随意改了不相关的代码。本质原因是 AI 在做决策时缺乏结构约束，每次的"心情"不同。

Archon 的解法：将开发流程编码为 YAML 工作流文件（存放在 `.archon/workflows/`），AI 的智能填充每个步骤的具体内容，但**结构是确定的、人控制的**。

## b. 核心架构与技术细节

**技术栈**：Bun（JS运行时）+ TypeScript + SQLite/PostgreSQL，Claude Code / Codex 作为 AI 执行引擎

**YAML 工作流示例**：
```yaml
nodes:
  - id: plan
    prompt: "探索代码库，制定实现计划"
  - id: implement
    depends_on: [plan]
    loop:
      prompt: "读取计划，实现下一个任务，运行验证"
      until: ALL_TASKS_COMPLETE
      fresh_context: true
  - id: approve
    depends_on: [run-tests]
    loop:
      prompt: "展示变更给用户审查"
      until: APPROVED
      interactive: true  # 暂停等待人类输入
  - id: create-pr
    depends_on: [approve]
    prompt: "推送变更并创建 Pull Request"
```

**工作流特性**：
- **隔离执行**：每个工作流运行获得独立 git worktree，5个任务可并行运行无冲突
- **Loop 节点**：AI 循环直到满足条件（`ALL_TASKS_COMPLETE`/`APPROVED`），内置 `fresh_context` 避免上下文污染
- **人类审批门控**：`interactive: true` 节点暂停等待人类输入
- **确定性+AI 混合**：bash 节点（运行测试）与 AI 节点（写代码）可混合编排

**17个内置工作流**：涵盖 issue 修复、功能开发、PR 审查（5个并行评审 Agent）、架构重构、测试循环、冲突解决等

**多平台支持**：Web UI / CLI / Telegram / Slack / GitHub Webhooks / Discord，全部汇聚到同一 Dashboard

## c. 竞品对比与生态站位

| 工具 | 工作流结构化 | 确定性 | 并行执行 | 人类审批门控 |
|------|-----------|--------|---------|------------|

Archon 是 n8n 在软件开发场景的垂直版本，与 Multica 互补（Multica 是团队协作层，Archon 是工作流引擎层）。

注意：原版 Archon（任务管理+RAG，Python）已归档到 `archive/v1-task-management-rag` 分支，当前版本是完全重写的工作流引擎。

## d. 开发者反馈与局限性

**正面**：
- git worktree 隔离方案非常工程化，解决了多任务并发执行的干净环境问题
- YAML 工作流是声明式的，可 Git 管理，团队共享同一套开发流程
- 17个内置工作流覆盖了 95% 的日常开发场景

**局限性**：
- 强依赖 Claude Code（`CLAUDE_BIN_PATH`），Codex 支持有限
- Loop 节点的 `until: ALL_TASKS_COMPLETE` 判断机制不透明，失败时可能无限循环
- 工作流 YAML 语法需要学习，团队采用有学习成本
- 多平台消息汇聚（Telegram+Slack+Discord+GitHub）的 Webhook 配置较为繁琐
- 原版 Archon 用户迁移到新版本需要完全重学（两个版本定位完全不同）
