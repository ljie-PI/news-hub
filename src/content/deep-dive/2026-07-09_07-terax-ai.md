# crynta/terax-ai

## 定位与痛点剖析
terax-ai（Terax）定位为「终端优先的 AI 原生开发工作区」，安装体积仅约 7-8 MB，强调无遥测、无账号。它瞄准现有终端与 IDE 的痛点：要么臃肿、要么缺乏内建的 agentic AI 能力。Terax 把多标签终端、代码编辑器、文件浏览器、带 git graph 的源码管理与 Web 预览整合进一个轻量原生应用，API 密钥写入系统 keychain 而不落盘，兼顾隐私与开箱即用。

## 核心架构与技术细节
应用基于 Tauri 2 + Rust 与 React 19 构建，终端采用 xterm.js WebGL 渲染、通过 portable-pty 提供原生 PTY 后端，编辑器为 CodeMirror 6 并支持 Vim 模式与逐块接受的 AI edit diff。AI 侧栏采用 Vercel AI SDK v6，BYOK 支持 OpenAI、Anthropic、Gemini、Groq、xAI、DeepSeek 等，本地可接 LM Studio、MLX、Ollama，具备计划模式、子 agent 与基于 `TERAX.md` 的项目记忆。

## 竞品对比与生态站位
相较 Electron 系的 VS Code 或商业化的 Warp、Cursor，Terax 以 Tauri 原生栈换取 7 MB 级极小体积与更低资源占用，主打开源、本地模型友好。它在 Linux 生态提供 AUR、Nix flake、AppImage 等多渠道分发，站位为「隐私优先、可自带模型的轻量终端 IDE」，吸引反感云端绑定与臃肿工具链的开发者。

## 开发者反馈与局限性
真实 issue 显示体验仍不稳定：#974 反馈连接 LLM 提供商后 AI agent 无法工作；#965 报告 history 命令中 backspace/delete 无法删除字符；#962、#961 两个 PR 分别修复终端刚打开时丢按键、IME 输入重音重复累加的问题。功能诉求包括 #972 请求 Claude Code slash 命令、#970 希望直接编辑 .md 文件、#969 请求新增保存快捷键或自定义快捷键、#966 希望 AI 面板连接提供商的横幅可折叠。此外 Windows 端尚未代码签名，首次启动会被系统拦截，当前仍有约 358 个未决 issue，打磨空间不小。
