# stagewise：能"看见"你应用的开源编码 Agent

## 事件背景

stagewise 是一款开源编码 Agent，核心差异化在于它能在自己的浏览器环境中运行，直接读取 DOM 和控制台输出，并编辑对应的源代码。产品在 Product Hunt 获得 174 票和 5 星满分评价，排名第五。

当前的 AI 编码工具（Cursor、Copilot 等）主要基于代码文本进行理解和生成，但无法直接"看到"代码运行后的实际效果。stagewise 通过将浏览器环境整合进 Agent 工作流，弥合了代码与渲染结果之间的鸿沟。

## 核心观点/产品机制

stagewise 的技术亮点：

- **自有浏览器环境**：Agent 在独立浏览器中运行你的应用，看到与用户相同的界面
- **DOM + Console 感知**：不仅读代码，还读运行时的 DOM 结构和 console 输出
- **代码 ↔ 视觉双向映射**：看到界面问题后，直接定位并编辑对应的源代码
- **模型无关**：支持任何 LLM，用户自带 API Key，保持完全控制
- **开源**：代码开放，社区可贡献和审计

## 社区热议与争议点

1. **"视觉感知"是否是编码 Agent 的必需**：支持者认为前端开发中很多 bug 只有看到渲染结果才能发现，DOM 感知是质的飞跃。反对者认为对于后端开发毫无意义，增加了不必要的复杂度。

2. **开源 vs 商业模型之争**：作为开源项目，stagewise 如何持续维护和发展？社区讨论了开源 AI 编码工具的商业模式——是走 Cursor 的 SaaS 路线还是保持纯开源？

3. **安全隔离的可靠性**：Agent 拥有浏览器访问权限引发了安全担忧——如果应用涉及敏感数据，Agent 的浏览器环境如何保证数据不泄露？

4. **与 Anthropic Computer Use 的比较**：有用户将 stagewise 与 Claude 的 Computer Use 功能对比。stagewise 更专注于 Web 开发场景，粒度更细（DOM 级别），而 Computer Use 是通用的桌面操作。

## 行业影响与未来展望

stagewise 代表了编码 Agent 发展的一个重要方向：**从"理解代码"到"理解代码运行的结果"**。这对前端和全栈开发有重大意义。

预计未来的编码 Agent 将标配某种形式的"运行时感知"——不仅理解代码文本，还能观察代码的执行效果、测试结果和用户界面。stagewise 的开源特性也可能推动这一能力成为行业标准。
