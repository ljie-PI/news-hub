# Nutlope/hallmark

**GitHub Repo**: https://github.com/Nutlope/hallmark
**语言**: CSS / HTML / JavaScript · MIT 许可 · ⭐ 约 4.5k

## 定位与痛点剖析

Hallmark 是一款为 Claude Code、Cursor、Codex 等 AI 编码助手设计的"反 AI 味"（anti-AI-slop）设计技能（skill），由 Together AI 出品。它解决的痛点极其精准：当下用 AI 生成的网页 UI 高度同质化——千篇一律的居中英雄区、渐变紫、圆角卡片、Inter 字体，一眼就能看出是"AI 做的"。Hallmark 通过一套强制规则，让 AI 在生成界面时跳出训练分布里的"默认审美"，使两个不同 brief 生成的页面呈现真正不同的结构与气质，而非同一模板的换色版本。

## 核心架构与技术细节

Hallmark 的核心是 `SKILL.md` 规则集加 `references/` 参考库。工作机制分几步：先为 brief 选择一种"宏观结构"（macrostructure），再套上 20 种主题之一，随后运行 **57 项 slop-test 门禁**逐条排查 AI 陈词滥调，最后在输出前做一次自我批判（pre-emit self-critique）。它提供四个"动词"：默认（构建新 UI）、`audit`（对现有代码打分并给出问题清单、不改动）、`redesign`（保留文案/信息架构/品牌，用不同"指纹"重建结构）、`study`（从欣赏的设计中提取 DNA——宏观结构、字体配对、色彩锚点，可导出便携的 `design.md` 交接给其他 AI 工具，但拒绝像素级克隆和盗用付费模板）。仓库主要是 CSS/HTML 演示站点，规则本身与工具无关。

## 竞品对比与生态站位

Hallmark 处在"AI 设计质量增强"这一新兴细分，与各类 shadcn/ui 组件库、v0、以及零散的"design system prompt"相比，它不提供组件而提供**约束与品味**，是叠加在生成流程上的质控层。它借力当前爆发的 agent skills 生态（Claude Code / Codex skills），可直接安装到 `~/.claude/skills` 等目录，分发门槛低。

## 开发者反馈与局限性

4.5k stars 且快速上升，反映开发者对"AI slop 设计"的普遍厌倦。局限在于：审美是主观的，57 条门禁强推的"去默认化"可能对某些品牌反而过度、产出难以预期；规则效果高度依赖底层模型对指令的遵循度，弱模型可能无法真正跳出默认分布；且它偏重视觉指纹，对可访问性、性能等工程维度覆盖有限。
