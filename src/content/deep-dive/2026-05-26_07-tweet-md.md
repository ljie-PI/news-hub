# tweet.md：把 X 帖子转换成干净的 Markdown

## 一句话定位

**X posts as clean Markdown** —— 一个把 Twitter/X 上的帖子（含线程、引用、媒体）一键转成结构化、可读、可存档的 Markdown 的小工具。

## 背景：为什么需要它

X 的内容长期处于"半封闭"状态：原生分享体验差、API 收费极贵、嵌入受限、长线程阅读体验糟糕。同时，越来越多高质量的技术、AI、创业内容首发在 X 上（thread 形式），用户有强烈的归档、阅读、整理、二次分发需求。这催生了一批 thread 工具：Thread Reader App（阅读化）、Typefully（写作端）、以及大量"截图美化"工具。tweet.md 走的是另一条路：**作为 Markdown 原生格式输出**，方便接入 Obsidian、Notion、博客、RSS、LLM workflow。

## 结构 B：产品形态

1. **输入**：粘贴 X URL（单条或线程），或浏览器扩展一键抓取。
2. **解析**：抓取帖子文本、作者信息、时间戳、回复链、引用嵌套、媒体（图片 / 视频链接）、外链卡片等。
3. **输出**：标准 Markdown，含 frontmatter（作者、时间、URL）、正文（保留链接、提及、emoji）、媒体（以 `![]()` 嵌入或本地下载）、线程层级（缩进或分段）。
4. **可能附加**：批量导出、API、与 Obsidian/Readwise/Notion 直接集成，或导出为 RSS。

## 适用人群

- **PKM 重度用户**：Obsidian / Logseq / Notion 用户希望把 X 上的精华内容沉淀到自己的知识库。
- **写作者 & 研究者**：引用 X 内容到博客或论文，干净 Markdown 比截图更可检索。
- **LLM / Agent 开发者**：把 X 内容作为输入喂给 LLM 时，Markdown 比原始 HTML 干净得多，token 利用更高效。
- **存档党**：对抗 X 上经常出现的删帖、账号封禁、可见性变化。

## 风险与限制

- **API 成本与合规**：X 自 2023 年起对 API 实施高额收费且条款严格，工具若不通过官方 API，可能依赖 scraping，存在被限流 / 法律风险。
- **媒体托管**：图片视频若仅引用原始 URL，X 一旦失效则失效；本地化又涉及存储与版权。
- **替代品众多**：Thread Reader App、x-to-md 类开源脚本、各种 bookmarklet 都在做类似事情，需要在体验细节胜出。

## 价值评估

定位极其清晰、ICP 明确、产品复杂度可控，是典型"小而美"工具型 PH 项目。增长依赖 PKM 社区口碑与开发者 workflow 集成。商业模式可能走 freemium（免费单条 + 付费批量 / API / 集成）。短期不会成为大公司，但服务一个真实需求，值得书签存一份。
