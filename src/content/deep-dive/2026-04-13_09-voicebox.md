# voicebox — 开源本地语音合成工作室

**仓库**: https://github.com/jamiepine/voicebox  
**今日新增 Star**: +408  
**日期**: 2026-04-13

## 定位与痛点剖析

ElevenLabs 的崛起证明了高质量 AI 语音合成的强大需求，但它有两个显著问题：**隐私和成本**。所有声音数据和生成请求都上传到云端，月费随用量增长，免费层严格受限。对于播客制作者、游戏开发者、有声书作者、隐私敏感用户来说，一个完全本地运行、功能完整的替代品是刚需。

`voicebox` 是 **ElevenLabs 的开源本地平替**——它将声音克隆、多引擎 TTS、音效后处理、多轨故事编辑器全部集成进一个桌面应用，所有计算在本地完成，数据不离开你的机器。

## 核心架构与技术细节

### 五大 TTS 引擎

voicebox 最大的技术亮点是**可切换的多引擎架构**，每个引擎针对不同场景：

| 引擎 | 参数量 | 语言数 | 特点 |
|------|--------|--------|------|
| **Qwen3-TTS** | 0.6B / 1.7B | 10 | 高质量多语言克隆，支持语音指令（"慢速说"、"低声说"） |
| **LuxTTS** | 轻量级 | 英语 | ~1GB VRAM，48kHz 输出，CPU 上 150x 实时速度 |
| **Chatterbox Multilingual** | - | 23 | 最广语言覆盖（含阿拉伯语、日语、印地语、斯瓦希里语等） |
| **Chatterbox Turbo** | 350M | 英语 | 快速，支持副语言情感标签 |
| **TADA（HumeAI）** | 1B / 3B | 10 | 文本-声学双对齐，可生成 700s+ 连贯长音频 |

### 副语言情感标签（Paralinguistic Tags）

Chatterbox Turbo 支持在文本中嵌入情感/动作标签，模型会在 TTS 输出中合成这些元素：

```
[laugh] [chuckle] [gasp] [cough] [sigh] [groan] [sniff] [shush] [clear throat]
```

例如：`"好吧[sigh]，我们就这么决定了。[laugh]"`

这是区别于普通 TTS 的重大差异，让生成的语音具有真实的情感质感。

### 后处理音效引擎

基于 Spotify 的 `pedalboard` Python 库，提供 8 种实时音效：
- Pitch Shift（±12 半音）
- Reverb（可配置房间大小/衰减/干湿比）
- Delay（回声）
- Chorus/Flanger（调制延迟）
- Compressor（动态范围压缩）
- Gain、High-Pass Filter、Low-Pass Filter

内置 4 个预设（Robotic、Radio、Echo Chamber、Deep Voice），支持自定义预设并可绑定到声音 Profile。

### 无限长度生成（Auto-Chunking）

文本在句子边界自动切分，各 chunk 独立生成后用 **Crossfade** 拼接。参数：
- 自动切分上限：100–5,000 字符（可配置）
- Crossfade 时长：0–200ms
- 最大文本长度：50,000 字符
- 智能分割：识别缩写、CJK 标点、情感标签

### 版本管理与溯源

每次生成支持多版本：
- **Original**：原始 TTS 输出，永久保留
- **Effects 版本**：不同效果链的结果
- **Takes**：不同 seed 的重新生成
- **溯源追踪**：每个版本记录其生成链路

### 技术栈

- **桌面框架**：**Tauri（Rust）**，非 Electron，原生性能，包体更小
- **前端**：现代 Web 技术（React 系）
- **音效处理**：Python `pedalboard`（Spotify 开源）
- **硬件加速**：macOS（MLX/Metal）、Windows（CUDA）、Linux（AMD ROCm、Intel Arc）
- **API**：内置 REST API，方便集成到外部项目

### 平台支持

| 平台 | 下载格式 |
|------|---------|
| macOS Apple Silicon | DMG |
| macOS Intel | DMG |
| Windows | MSI |
| Docker | `docker compose up` |
| Linux | 需从源码构建 |

## 开发者反馈与局限性

**正面**：
- Tauri 架构比 Electron 方案轻量得多，用户报告启动快
- 多引擎切换体验流畅
- 情感标签功能被播客用户高度评价

**局限**：
1. **Linux 无预编译包**：需从源码构建，对普通用户门槛高
2. **硬件要求**：某些引擎（如 TADA 3B）需要足够的 VRAM，低端机器体验差
3. **Qwen3-TTS 语言仅 10 种**：最高质量引擎不支持全部 23 语言
4. **TADA 引擎依赖 HumeAI**：虽然本地运行，但模型来自商业公司，未来许可可能变化
5. **早期项目**：部分功能仍在迭代，API 不稳定
