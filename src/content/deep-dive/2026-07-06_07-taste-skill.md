# Leonxlnx/taste-skill

## 定位与痛点剖析
Taste Skill 自称「反平庸前端框架」，本质是一组可移植的 Agent Skills（SKILL.md 指令文件），用来纠正 AI 生成前端界面千篇一律、模板味重的问题。它面向用 Codex、Cursor、Claude Code 等智能体做前端的开发者，目标是让 AI 输出更强的排版、留白、动效与视觉层次，而非生成千篇一律的样板 UI。

## 核心架构与技术细节
项目并非传统代码库，而是一批 JavaScript 生态下的技能包，通过 npx skills add 安装到智能体。核心 taste-skill（v2 实验版）会读取需求、推断设计语言，并用 VARIANCE、MOTION、DENSITY 三个档位调节布局激进度、动效强度与信息密度，内置 GSAP 动画骨架与禁用 em-dash 等反重复规则。仓库还含 minimalist、brutalist、soft 及图像生成等多种变体技能。

## 竞品对比与生态站位
与通用 AI 设计提示词相比，它胜在细分变体多、档位可调与基于研究的反套路规则，且跨 React、Vue、Svelte 框架无关。它兼容 Vercel 的 agent-skills 规范，卡位在提升 AI 编码产物审美这一新兴赛道，属于可复用的技能资产而非运行时工具，安装后即插即用。

## 开发者反馈与局限性
项目号称当日新增约 850 星，社区热度极高，但需理性看待，星标增速与实际成熟度未必匹配。Issue 多为社区预设、渐进式披露拆分、快速试用链接等建议，也混入无关灌水贴，反映其更多依赖话题声量，实际效果仍取决于底层模型能力。
