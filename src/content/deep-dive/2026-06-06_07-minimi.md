# Minimi

## 一、事件背景

Minimi 是 Shram 团队在 Product Hunt 推出的最新作品，定位为"Claude 的环境记忆（ambient memory）"。Shram 此前已两次冲上 PH 当日第二名（Shram 1.0 在 2024 年 7 月、Shram 2.0 在 2025 年 2 月），积累了 1.5K 关注者基础。本次发布延续团队对"AI 对话上下文缺失"问题的关注：用户每天打开 Claude 或 ChatGPT，都要重新解释自己的项目、角色、上下文——Jay Gadekar 在帖子中直言"我们的 AI 助手每天都在失忆，为什么我们能容忍这件事？"Minimi 正是要为 Claude 提供贯穿整台 Mac 的长期记忆基础设施，面向开发者、知识工作者与重度 AI 助手用户。

## 二、核心观点 / 产品机制

Minimi 是一款 Mac 原生应用，能够在用户授权下静默捕捉电脑上的几乎所有上下文——浏览器标签、文档编辑、视频通话、Slack 对话——并将整理后的语境实时喂给 Claude。用户无需手动 brief，直接发问即可，例如"是谁把那张 bug 截图发给我的？"或"昨天会议我们达成了什么决议？"。技术层面，Minimi 在设备端运行本地向量数据库，所有捕获数据均不上云，并提供按 App 粒度的暂停、屏蔽与卸载即清空机制。性能上，团队在 BEAM 基准（1M / 10M token 规模）上跑出 54% 的准确率，相较先前 SOTA 的 36% 提升约 50%，并在 LongMemEval 长对话检索上也有亮眼表现。当前仅提供 Mac 版本，PH 社区可领取一个月免费体验。

## 三、社区热议与争议点

PH 评论区出现几条具体讨论：(1) 用户 Ozan 担忧"被动捕获一切会让 Claude 拿到大量自信但无关的噪音"，Vineet Gupta 回应"价值在于挑选呈现什么，而非记录一切"，因此团队选择以 BEAM 等准确率指标而非召回率指标做基准；(2) Zaid Mallik 追问"记忆质量随时间下降"的问题，Ojasvika Sahu 回答 Minimi 将"更新、矛盾、时间序"作为核心行为而非补丁；(3) Tom Palmer 询问暂停/撤销机制——团队明确"暂停期间不产生任何记忆"；(4) Anna Ludwinowski 询问 Windows 版本，团队解释 Mac 的 Accessibility 框架在 Windows 上不够稳定，故采取 Mac 优先策略。

## 四、行业影响与未来展望

Minimi 代表"环境式记忆基础设施"的新方向，弥补 Claude、ChatGPT 原生记忆功能的不足。若 BEAM 等准确率指标在真实工作流中能持续兑现，它有望成为 Claude 重度用户的默认伴侣应用，并向 Windows 与选择性记忆删除等方向扩展。
