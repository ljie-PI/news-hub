# github/spec-kit

## 定位与痛点剖析

Spec Kit 是 GitHub 官方推出的"规格驱动开发"（Spec-Driven Development, SDD）工具包，用于把 AI 编码助手（Copilot、Claude Code、Cursor 等）的工作方式从"凭一句提示词生成代码"升级为"先写清晰规格、再由 AI 按规格实现"。它瞄准的痛点非常具体：当下 vibe coding 模式下，开发者常常得到看似能跑但结构混乱、需求漂移、难以维护的代码；缺乏明确规格导致 AI 反复返工、上下文丢失。Spec Kit 通过引入结构化的规格文档（PRD 式）、任务拆解与执行流程，让 AI 的产出可控、可审、可迭代。

## 核心架构与技术细节

项目主体是 Python 编写的 `specify_cli`（占比 94.8%），辅以 Shell 与 PowerShell 脚本实现跨平台。核心是一套模板体系（`templates/`）与工作流（`workflows/`），把开发拆成规格化阶段：建立项目"宪法"（`.specify/memory`，即项目级约束与原则）、编写 specification、生成实现计划、拆解 tasks、再驱动 AI 执行。它以斜杠命令形式集成进多种 AI 编码代理，并提供 presets、extensions 与 integrations 扩展点。项目迭代极快，已发布 182 个 release，最新为 0.12.6（2026 年 7 月）。

## 竞品对比与生态站位

Spec Kit 与 BMAD-Method、Kiro（AWS 的规格驱动 IDE）、Taskmaster 等"AI 工程化/规格驱动"工具处于同一赛道。它最大的护城河是 GitHub 官方背书与对 Copilot 生态的原生集成，能直接触达海量 GitHub 用户。相较纯 IDE 方案，Spec Kit 是工具无关的 CLI，可搭配任意 AI 代理，灵活性更高；相较轻量 prompt 框架，它更强调完整的规格—计划—任务—实现闭环。

## 开发者反馈与局限性

近 12 万 stars 说明规格驱动开发正成为社区共识。但 SDD 也带来前期成本：写规格、维护"宪法"需要纪律，小型或探索性项目可能觉得流程过重、收益不明显。工具仍在高速演进（版本号密集迭代），API 与最佳实践尚未稳定，学习曲线和文档跟进是常见反馈点。此外其效果高度依赖底层 AI 模型能力，规格再清晰也无法弥补模型本身的理解偏差。
