# google-labs-code/design.md 深度解析

## 定位与痛点剖析

`google-labs-code/design.md` 的定位不是又一个组件库，而是一份给编码智能体读取的视觉身份规格。项目描述强调：`DESIGN.md` 通过持久、结构化的方式，让智能体理解一个设计系统。它要解决的核心痛点，是当前 AI 写前端时经常“只会按一次性提示生成页面”，缺少跨会话、跨页面、跨仓库可复用的审美约束。传统 `README`、`AGENTS.md` 或 `CLAUDE.md` 多半说明工程规则，却很少告诉模型为什么按钮应该是某种颜色、某种留白为何重要、哪些视觉手法应避免。该项目把设计上下文变成版本化文本：既能被人审阅，也能被工具校验，还能随代码一起演进。仓库目前约 1.7 万星、1.5 千余 fork，本周期新增 504 星，说明“给 Agent 一份设计契约”正在成为前端自动化的重要共识。

## 核心架构与技术细节

`DESIGN.md` 文件采用“两层结构”：顶部 YAML front matter 放机器可读的设计 token，正文 Markdown 放人类可读的设计意图。README 明确区分：token 是规范值，prose 解释这些值为什么存在、如何应用。规格支持颜色、尺寸、token 引用和 typography 对象；typography 可包含 `fontFamily`、`fontSize`、`fontWeight`、`lineHeight`、`letterSpacing`、`fontFeature`、`fontVariation` 等字段。正文使用固定的二级标题顺序：Overview、Colors、Typography、Layout、Elevation & Depth、Shapes、Components、Do's and Don'ts；部分章节可省略，但出现时需遵守顺序。组件 token 支持 `backgroundColor`、`textColor`、`typography`、`rounded`、`padding`、`size`、`height`、`width` 等属性，hover、active、pressed 等状态建议作为独立组件项表达。工具层面，CLI 包发布为 `@google/design.md`，可执行 lint、diff、export、spec 等任务：lint 能检查结构、断裂引用和 WCAG 对比度；diff 能比较两个设计系统的 token 与文本回归；export 可面向 JSON、Tailwind 或 DTCG 等格式转换。仓库本身是 TypeScript/Bun/Turbo monorepo，说明其目标不仅是文档规范，也包含可集成的自动化工具链。

## 竞品对比与生态站位

与 W3C Design Token Format、Style Dictionary 或 Tailwind 配置相比，`DESIGN.md` 的差异在于它不只保存数值，而把“叙事意图”提升为一等公民。Design Token 生态擅长跨平台同步颜色、间距和字体，但很难表达“这个红色只能用于关键行动，不能用于元数据”这样的审美边界；`DESIGN.md` 则用 prose 补上语义、情绪和禁用规则。与 Figma 变量或设计系统站点相比，它牺牲了可视化编辑体验，却获得了纯文本、可 diff、可 code review、可被 Agent 直接消费的优势。与 `AGENTS.md`、`CLAUDE.md` 类工程上下文文件相比，它面向的是视觉决策层，可成为仓库中“设计上下文”的固定入口。因此它更像 Agent-native 时代的设计契约：轻量、可提交、可校验，并能让不同模型或不同工具在同一品牌语言下工作。

## 开发者反馈与局限性

从 issue 列表看，社区关注点集中在规格边界尚未稳定：有人提出 iconography 章节、结构化 `shadows`/`elevation` token、motion token、色盲对比度规则、组件变体支持、基础字号与段落最大宽度、单项目多份 `DESIGN.md` 的发现规则，以及用 `omitted` 声明“刻意缺省”以抑制警告。这些问题说明项目已经击中了真实需求，但也暴露 alpha 规范的不足：它当前更擅长颜色、排版、间距、圆角和基础组件，对动效、图标、复杂响应式布局、无障碍高级规则和多品牌场景的覆盖仍有限。另一个实际问题来自 CLI 分发：npm registry 配置错误可能导致 `ENOVERSIONS`，Windows/PowerShell 下 `design.md` 可执行名还可能和 `.md` 文件关联冲突，官方建议使用无点别名 `designmd`。因此，团队若要引入它，最好先把它当作“设计意图源文件”和 Agent 约束层，而不是完整替代 Figma、组件库或现有 token pipeline。
