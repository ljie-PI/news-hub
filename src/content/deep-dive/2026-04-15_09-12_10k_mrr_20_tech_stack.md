# 我用每月 $20 的技术栈运营多家月收入 $10K 的公司

**得分：954 | 评论：503 | 作者：tradertef**

---

## 事件背景

独立开发者 Steve Hanov（websequencediagrams.com 作者）分享了自己如何用极简技术栈同时运营多家月收入超过 $10K 的公司，每月基础设施成本仅 $20 左右。这篇文章击中了 HN 社区对过度工程化和 VC 驱动式创业的反思情绪，在 503 条评论中引发了关于"正确规模"工程实践的深度讨论。文章本人表示曾被多个投资人 pitch night 拒绝，理由是"你还需要融资干什么？"

---

## 核心观点 / 产品机制

**技术栈核心原则**：

1. **精简服务器**：使用 Linode 或 DigitalOcean 的单一 VPS，$5-10/月，1GB RAM 足够——不需要 AWS EKS 集群、RDS 实例、NAT Gateway 等。作者明确说"AWS 的控制台是一个为提取账单升级而设计的迷宫"。

2. **Go 后端**：选择 Go 而非 Python/Ruby，理由：
   - 性能远超，不浪费内存在解释器启动和 worker 管理
   - 严格类型系统
   - 2026 年 LLM 非常容易推理 Go 代码
   - 部署极简：编译成单一静态链接二进制，`scp` 到服务器直接运行，无 pip 依赖地狱

3. **本地 AI 替代 API**：用淘到的 $900 二手 RTX 3090（24GB VRAM）跑 vLLM，彻底避免 AI API 账单。升级路径：Ollama（原型验证） → vLLM（生产，PagedAttention 批处理优化）→ Transformer Lab（微调/预训练）。

4. **自建工具**：
   - `laconic`：针对 8K 上下文窗口优化的 agentic 研究器，像 OS 虚拟内存一样管理 LLM 上下文
   - `llmhub`：将任何 LLM 抽象为统一的 provider/endpoint/apikey 组合

## 社区热议与争议点

**支持方：技术栈规模与业务规模应匹配**
- 大量评论者表达了对"过度工程化"的共鸣，认为 $10K MRR 的业务根本不需要 Kubernetes 集群。

**最实质性的技术争论：Swap 文件**
- **hackingonempty** 指出：应该总是使用 swap 文件/分区，即使不想要真正的交换——因为"冷页"（cold pages）如果没有 swap 空间就无法被换出，会白白占用内存。
- **berkes** 补充了 nuance：在单服务 + 编排重启的场景下，宁愿让进程 OOM crash 立即重启，而不是在降级状态下运行——这是 swap 使用哲学的真正分歧所在。

**争议点：AI 成本被低估**
- **komat** 指出文章没有提到 Claude Code 或类似 Coding Agent 的成本
- **PhilippGille** 澄清：作者特意使用 GitHub Copilot，因为微软按请求而非按 token 计费

**对融资文化的反思**：评论区出现了大量"创业不一定要 VC"的讨论，许多人分享了类似的精益运营经历，认为 VC 模式适合特定类型的公司，但对于可盈利的 SaaS 小产品是完全错误的路径。

## 行业影响与未来展望

这篇文章的走红反映了 2026 年独立开发者生态的重要趋势：AI 工具（本地 LLM + Copilot）正在将单人开发者的生产力提升到前所未有的水平，使得"一个人运营多家盈利公司"成为现实。

技术选型建议（Go + 单 VPS + 本地 AI）为追求极简运营的创始人提供了一份可复制的实践指南。随着 vLLM 和消费级 GPU 的普及，"自托管 AI"的成本门槛将继续降低，类似的精益 AI 产品将大量涌现。对投资机构而言，这也是一个信号：越来越多的优质小型 SaaS 业务永远不需要也不想要 VC 投资。
