# 14 · TheCraigHewitt/seomachine

> ⭐ 2,562 stars / week | 6,198 total | 2026-04-15

## a. 定位与痛点剖析

**SEO Machine** 是一个专门用于生产 SEO 优化长文的 Claude Code 工作空间，定位是"内容营销团队的 AI 生产力工具"。

核心痛点：
1. **内容营销成本高**：雇用专职 SEO 内容写手月薪数千美元，外包每篇文章也要几百美元，但质量参差不齐
2. **AI 写的内容太通用**：直接让 ChatGPT/Claude 写博客，输出的是"AI 腔"内容，缺乏品牌声音、没有竞争定位意识、不符合真实 SEO 标准
3. **研究与写作割裂**：内容团队往往手动研究竞品内容、关键词密度、SERP 分析，效率低
4. **无数据闭环**：写了文章但没有基于 GA4/Google Search Console 数据优化的机制

SEO Machine 通过 Claude Code 的 `/research → /write → /optimize → /publish` 完整流水线，配合 GA4/GSC/DataForSEO 数据集成，解决了上述问题。

## b. 核心架构与技术细节

**技术栈**：Claude Code Workspace + Python（NLP/分析模块）+ WordPress REST API

**目录结构逻辑**：
- `.claude/commands/`：自定义命令（/research, /write, /rewrite等）
- `.claude/agents/`：专门化分析 Agent（SEO 优化/Meta 生成/内链分析等）
- `.claude/skills/`：26个营销技能文件
- `data_sources/modules/`：Python 分析模块（关键词密度/可读性/竞品对比等）
- `context/`：品牌声音/写作示例/关键词/竞品分析（**必须手动填写**）

**5个 Python 分析模块**：
1. `search_intent_analyzer.py`：搜索意图分类（信息/导航/交易/商业）
2. `keyword_analyzer.py`：关键词密度+聚类+LSI（潜在语义索引）关键词
3. `seo_quality_rater.py`：0-100 综合评分（内容/关键词/Meta/结构/链接/可读性）
4. `content_length_comparator.py`：爬取 SERP 前10竞品统计词数，提供"需写多少字"的数据依据
5. `readability_scorer.py`：Flesch 阅读难度 + 被动语态检测 + 句子复杂度

**26个营销技能**：覆盖文案写作/CRO（转化率优化）/A-B测试/邮件序列/定价策略/SEO审计/程序化SEO等

**WordPress 集成**：通过 REST API + Yoast SEO MU-plugin，`/publish-draft` 一键发布含 SEO metadata 的文章

## d. 开发者反馈与局限性

**正面**：
- 开发者实际用于自己公司（Castos，播客托管 SaaS），有真实落地案例
- 26个营销技能覆盖了内容团队日常 95% 的任务类型
- WordPress 发布集成对非技术内容编辑友好

**局限性**：
- **Context 文件必须手动填写**：brand-voice.md/writing-examples.md 等如果填写粗糙，输出质量会显著下降，这是最大的门槛
- DataForSEO 需要付费 API，增加运营成本
- Python 依赖（nltk/scikit-learn等）安装可能遇到环境问题
- WordPress 集成需要安装 MU-plugin，对没有技术背景的内容运营有门槛
- 无 GUI，纯 Claude Code 界面，不适合非开发者用户
