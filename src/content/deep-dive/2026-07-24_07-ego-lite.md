## 一、定位与痛点

ego (lite) 由 citrolabs 开源，定位为「你和你的 AI Agent 可以并行工作的最佳浏览器」。现有的 browser-use、Vercel agent-browser 等本质是浏览器自动化框架：需要另起一个浏览器来驱动，登录态难以干净继承，人与 Agent 还会争抢同一批标签页。ego lite 的解法是——一个从设计之初就为「人机共享」而生的浏览器，Agent 在各自独立的 Space 中跑任务，你的标签页始终属于你。

## 二、核心架构与技术细节

ego lite 主要由 JavaScript（约 58%）与 TypeScript（约 41%）实现。其核心是「Code base, not CLI base」：浏览器能力被封装成 Agent 可直接调用的 JavaScript 函数，Agent 把多步任务组合成一次代码输出，而非「调两条命令、看结果、再调两条」的循环。连接层 `ego-browser` 向任意 Agent CLI（Claude Code、Codex、Cursor 等）暴露 snapshot、fill、click、wait、navigate、capture 等页面内工具。每个 Agent 拥有完全隔离的 Space，支持多任务并行；得益于内核级定制，其页面 Snapshot 能可靠处理深层嵌套 iframe。

## 三、竞品对比与生态站位

官方对比表将其与 Browser-Use、agent-browser、ChatGPT Atlas、Perplexity Comet 并列：相比自动化框架，ego lite 自带浏览器并能干净继承 Chrome 数据；相比 AI 浏览器只能用内置 Agent，它可由任意外部 Agent 驱动，且支持并行、本地存储与免费使用。

## 四、开发者反馈与局限性

项目约 665 star、38 fork、221 次提交（自报口径）。官方称复杂任务较传统 CLI 方案「最快 2.5× 提速、更少 token」（营销数据，自报口径）。局限也明显：目前仅支持 macOS，Windows/Linux 尚在路线图；「经验积累」等特性标注为 coming soon，生态与成熟度仍处早期。
