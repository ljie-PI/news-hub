# 08 · siddharthvaddem/openscreen

> ⭐ 4,765 stars / week | 29,780 total | 2026-04-15

## a. 定位与痛点剖析

**OpenScreen** 是 Screen Studio 的开源免费替代品，专注于制作产品演示视频和操作截图教程。

核心痛点：Screen Studio 是 macOS 上最受开发者喜爱的录屏/演示工具，功能完善（自动缩放、运动模糊、精美背景），但价格 $29/月（或 $99 一次性）对于个人开发者、开源项目维护者来说是一笔持续支出。

OpenScreen 的价值主张很简单：**覆盖大多数人实际需求的功能子集，完全免费，无水印，商业可用（MIT 协议）**。

目标用户画像：
- 独立开发者展示个人项目
- 开源维护者制作 README 演示
- 初创公司节省工具成本
- 希望完全控制工具链的开发者

## b. 核心架构与技术细节

**技术栈**：Electron（跨平台）、React、TypeScript、Vite、PixiJS（图形渲染）、dnd-timeline

**已实现功能**：
- **录制**：指定窗口或全屏录制
- **缩放**：自动或手动缩放（可调深度、时长、位置）
- **音频**：麦克风 + 系统音频录制
- **裁剪**：遮盖录制内容的某部分
- **背景**：壁纸/纯色/渐变/自定义背景
- **运动模糊**：平滑的平移和缩放效果
- **注释**：文字、箭头、图片标注
- **时间线编辑**：剪切片段，自定义各段速度
- **导出**：多种比例和分辨率

**系统音频限制**（平台差异）：
- macOS 13+（14.2+ 会有权限提示）
- Windows 开箱即用
- Linux 需 PipeWire（Ubuntu 22.04+ / Fedora 34+）
- macOS 12 及更早版本不支持系统音频

**macOS Gatekeeper 绕过**（未付费签名）：
```bash
xattr -rd com.apple.quarantine /Applications/Openscreen.app
```
然后在系统设置授予屏幕录制和无障碍权限。

## d. 开发者反馈与局限性

**正面**：
- 29.7k total stars，增势稳定，说明长期使用需求真实存在
- MIT 协议完全开放，商业使用无顾虑
- Electron+React 技术栈降低贡献门槛

**局限性**：
- README 明确标注"Beta 版本，可能有 bug"
- 作者自述"刚接触开源，不知道自己在干什么"，维护连续性有疑问
- 功能集与 Screen Studio 差距较大（无 AI 缩放决策、无鼠标特效等高级功能）
- macOS 未公证（无开发者证书），每次更新后都需要重新运行 `xattr` 命令
- Linux 系统音频支持依赖 PipeWire，老旧发行版需要额外配置
